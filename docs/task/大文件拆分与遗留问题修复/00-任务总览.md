## 任务目标

《供应商与模型扩展性改造》完成后的全仓库审查发现两类问题：一是若干遗留小问题（注册表 `subtitles` 场景无消费方、`providerKind` 字段命名误导、静音测试死代码分支、供应商泛化后残留的单供应商硬编码文案、`SettingsProviderPanel` 取 profile 方式不一致）；二是若干大文件职责混杂（后端 `transcription.rs` 944 行混装三条转写通路 + 音频编码 + HTTP 工具，`text_align.rs` 1259 行，`commands/asr.rs` 命令层与会话循环耦合；前端 `SubtitleEditor.tsx` 1067 行、听写 `controller.ts` 537 行多流程混杂）。

本次任务目标：**修复全部审查发现的遗留问题，并按解耦原则把上述大文件拆分为职责单一的模块，全程保持对外行为、公共 API 与持久化数据完全不变。**

**属于本次任务**：第一阶段列出的全部小问题修复；`providers/alibabacloud/transcription.rs`、`text_align.rs`、`commands/asr.rs`、`ui/src/components/transcription/SubtitleEditor.tsx`、`ui/src/features/dictation/controller.ts` 五个文件的拆分。

## 重要说明

- **行为不变是硬约束**：所有拆分任务都是纯重构，听写 / 实时字幕 / 录音识别 / 文稿对齐 / 字幕编辑的对外行为不允许变化。第一阶段两个修复任务允许的行为变化仅限任务文件中明确列出的项。
- **持久化兼容**：不得改动 `say-it-state.json` 的结构与读写逻辑；`providerKind` 改名只影响 `shared/asr-models.json`（编译期内嵌 + 打包内联的数据文件），不影响用户本地数据。
- **验证命令**（每个任务完成后按需执行）：`cd src-tauri && cargo check && cargo test`、`npx tsc --noEmit`、`npm run ui:build`。仓库当前没有 lint/format 脚本。
- 涉及麦克风、快捷键、注入、拖拽等真实交互的验证必须列出操作步骤交给用户实测，不要自行模拟（见 CLAUDE.md）。
