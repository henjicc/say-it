## 当前情况

单文件混装三类职责（行号为 2026-07-06 时点，1.2 完成后会有小幅偏移）：

- **Tauri 命令**：`start_asr_stream`（15，其中 74~282 是一整段约 210 行的 `tauri::async_runtime::spawn` 会话主循环闭包）、`asr_stream_push_chunk`（339）、`asr_stream_push_f32_chunk`（366）、`asr_stream_finish`（391）、`stop_asr_stream`（411）、`run_asr_silence_test`（429）。
- **会话循环支撑**：`AsrSession` 结构（8）、`send_or_queue_audio`（287）、`send_audio_message`（318）、`send_finish`（329），签名里带着冗长的 `SplitSink<WebSocketStream<...>>` 类型。
- **静音测试**：`run_realtime_silence_test`（442）、`asr_event_to_value`（1.2 中改名，512）。
- 会话循环已协议无关（只消费 `providers/connector.rs` 的 `AsrEvent` / `RealtimeAsrConnector`），与命令层的耦合点是 `emit_asr_stream_event`（来自 `commands/audio.rs`）和 `RuntimeState.asr_streams`。
- 命令以 `pub(crate)` 导出，`main.rs` 经 `commands::*`（执行时确认 `commands/mod.rs` 的组织方式）注册。

## 实施方案

`commands/asr.rs` → `commands/asr/` 目录：

```text
src-tauri/src/commands/asr/
├── mod.rs          # 六个 #[tauri::command] 定义（薄壳：解析供应商、建连接器、调 session/silence）+ pub(crate) 导出
├── session.rs      # AsrSession + 会话主循环（从 spawn 闭包提取为独立 async fn run_asr_session(...)）+ send_or_queue_audio / send_audio_message / send_finish + WebSocket writer 类型别名
└── silence_test.rs # run_realtime_silence_test + asr_event_to_value
```

要点：

- 会话循环从闭包提取为具名函数是本任务唯一的结构性变化：把闭包捕获的变量（writer/reader/rx/streams/app_handle/session_id/dsp 等）变成函数参数或参数结构体，循环体逐字保留；`start_asr_stream` 里仍是 `tauri::async_runtime::spawn(run_asr_session(...))`。
- 定义一个 `type WsWriter = SplitSink<WebSocketStream<MaybeTlsStream<TcpStream>>, Message>` 别名，消除三处重复的长签名（仅类型别名，不是新抽象）。
- 命令函数签名、名称、参数、返回值一律不变，保证前端 `lib/tauri.ts` 的 CMD 常量与 `main.rs` 注册零改动。
