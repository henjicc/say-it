## 实施方案

拆为同目录多文件，`controller.ts` 保留为对外门面（re-export + 顶层编排 `startDictation`/`toggleDictation`/`cancelDictation`），消费方 import 路径不变：

```text
ui/src/features/dictation/
├── controller.ts     # 对外门面：入口编排 + re-export 事件处理器（消费方零改动）
├── session.ts        # 会话状态集中为一个可变 session 对象 + 状态重置 + setDictationStatus/pushDictLog/clearDictLog
├── realtimeFlow.ts   # 实时听写：start/stop/finalize 调度、段结束处理
├── fileFlow.ts       # 文件听写：start/stop、原始块合并、转写结果提取
└── inject.ts         # runLocalProcessing + injectFinalText + finalizeDictation
```

要点：

- 模块级散落变量收敛为 `session.ts` 导出的单个可变对象（如 `export const dictSession = { sessionId: "", mode: null as ..., ... }` + `resetDictSession()`），各流程文件读写 `dictSession.xxx`——这是本任务唯一允许的结构性变化，字段与语义一一对应，不增不减。
- 函数体逐字搬运；`handleDictAsrEvent` 等事件处理器归属实时/文件流程文件，由 `controller.ts` re-export。
- 拆分边界以执行时的实际导出面 grep 结果为准，若 `startShortcutCapture`/`setInjectMethod` 等确在本文件，归入最贴近的新文件（快捷键捕获相关可考虑并入现有 `hotkeys.ts`，执行时判断改动更小者）。
