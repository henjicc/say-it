# 3.1 升级设置页与供应商配置

## 任务状态

- 状态：待开始
- 所属阶段：第三阶段-设置与复杂控件统一
- 前置任务：1.2
- 关联任务：3.2、4.1、4.2
- 是否可以并行：可以。可与 2.1、2.2、3.2 在 1.2 完成后并行，但需避免同时修改基础组件。

## 任务目标

升级设置页主结构、密钥与识别、启动设置、麦克风与提示音、外观页面，使设置区域与核心页面使用一致的桌面端视觉体系，并保留现有自动保存、折叠、高级参数和本地偏好逻辑。

## 当前情况

- `ui/src/views/SettingsView.tsx` 使用 tabs 切换密钥与识别、录音调整、启动设置、麦克风与提示音、外观。
- `ui/src/views/SettingsProviderPanel.tsx` 管理 Fun-ASR API Key、热词、高级参数和供应商状态。
- `ui/src/views/SettingsStartupPanel.tsx` 管理开机自启与静默启动。
- `ui/src/views/SettingsMicCuePanel.tsx` 管理输入设备、麦克风保活、提示音和自定义音频文件。
- `ui/src/views/SettingsAppearancePanel.tsx` 管理暗色/亮色和强调色。
- `ui/src/components/ui/Collapse.tsx` 已用于供应商配置和嵌套设置。
- 当前设置页视觉相对接近旧黑底卡片，tabs 与参考图一致性不足，折叠面板、输入框、按钮、说明文字仍需统一。

## 实施方案

保留设置页 tabs 和各子面板数据逻辑，统一设置页容器、tabs、折叠面板、API Key 输入区、高级参数表单、启动设置和外观预览。设置页不新增未确认入口，不改变 API Key 自动保存行为。

## 执行步骤

1. 调整 `SettingsView.tsx`：补充统一页头或设置页专用容器，保持 tabs key 不变。
2. 调整 `SettingsProviderPanel.tsx`：供应商折叠面板、API Key 输入、状态文案、热词和高级参数折叠区使用统一 token 和控件规格。
3. 保留 API Key 自动保存、防抖、显示/隐藏、远程读取真实 key、打开获取页面等逻辑不变。
4. 调整 `SettingsStartupPanel.tsx`：使用统一 section 和 checkbox/switch 布局，保留 Tauri 启动设置命令。
5. 调整 `SettingsMicCuePanel.tsx`：输入设备、保活秒数、提示音选择、自定义文件选择和试听按钮布局统一。
6. 调整 `SettingsAppearancePanel.tsx`：暗/亮色切换、强调色编辑、预览区、恢复默认按钮与 token 体系一致。
7. 记录人工验证步骤：API Key 输入/显示、折叠展开、启动设置开关、提示音文件选择、外观切换。
8. 运行 `npx tsc --noEmit` 和 `npm run ui:build`。
9. 更新本任务执行记录和 `00-任务总览.md`。
