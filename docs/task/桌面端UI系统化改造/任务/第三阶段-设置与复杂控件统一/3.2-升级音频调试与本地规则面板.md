# 3.2 升级音频调试与本地规则面板

## 任务状态

- 状态：待开始
- 所属阶段：第三阶段-设置与复杂控件统一
- 前置任务：1.2
- 关联任务：2.1、3.1、4.1、4.2
- 是否可以并行：可以。可与 2.1、2.2、3.1 在 1.2 完成后并行，但需避免同时修改 `Slider`、`Input`、`Button` 等基础组件。

## 任务目标

升级音频调试和本地文本处理规则等复杂高密度界面，使其在保留专业信息密度的同时，与新 UI 系统统一，并确保调试、规则编辑、预览和状态反馈清晰可用。

## 当前情况

- `ui/src/views/AudioView.tsx` 管理录音试听、A/B 播放、实时电平、波形 canvas、响度归一化、RNNoise 降噪、EQ 参数。
- `ui/src/views/LocalRulesPanel.tsx` 管理本地查找替换、正则规则、启用状态、规则排序、预览输入输出和错误提示。
- `ui/src/views/FunAsrHotwordsPanel.tsx` 管理热词列表，通常嵌入供应商配置中。
- `ui/src/components/ui/LogPanel.tsx` 用于调试日志显示。
- 这些界面控件多、密度高，当前样式更偏局部实现，输入框、列表、按钮、错误状态和 canvas 容器需要统一。

## 实施方案

以“桌面端高密度工具面板”为目标，不扩大功能范围。音频调试保留仪表和波形 canvas；本地规则保留当前规则数据结构与 worker 预览；热词面板只在样式需要时局部统一。

## 执行步骤

1. 调整 `AudioView.tsx`：录音试听操作区、实时电平、波形 canvas、参数滑块、说明文字、状态提示使用统一 section 和控件规格。
2. 保留 `lab.toggleRecord`、`lab.playOriginal`、`lab.playProcessed`、`lab.setCanvases`、`lab.paramChanged`、`lab.resetParams` 调用不变。
3. 调整 `LocalRulesPanel.tsx`：启用开关、查找替换列表、正则规则折叠编辑、排序按钮、删除按钮、试运行输入输出、错误提示统一视觉。
4. 保留 local rules 数据结构、worker 运行、防抖预览、规则排序和内置规则保护逻辑不变。
5. 检查 `FunAsrHotwordsPanel.tsx` 和 `LogPanel.tsx` 是否与新 UI 割裂；若明显割裂，做局部样式统一。
6. 记录人工验证步骤：录音/播放按钮、滑块调整、规则新增/删除/排序、正则错误提示、试运行预览。
7. 运行 `npx tsc --noEmit` 和 `npm run ui:build`。
8. 更新本任务执行记录和 `00-任务总览.md`。
