# 2.1 升级语音输入与实时字幕页面

## 任务状态

- 状态：待开始
- 所属阶段：第二阶段-核心页面布局升级
- 前置任务：1.2
- 关联任务：3.2、4.1、4.2
- 是否可以并行：可以。可与 2.2、3.1、3.2 在 1.2 完成后并行，但需避免同时修改同一个基础组件。

## 任务目标

按参考图升级语音输入和实时字幕两个高频页面，使其使用统一页头、分段页签、分组标题、两列表单、状态提示和桌面端控件布局，同时保留现有听写、快捷键、音频设备、注入方式、字幕预览和字幕运行逻辑。

## 当前情况

- `ui/src/views/DictationView.tsx` 负责语音输入页签容器，包括通用设置、文本处理、调试。
- `ui/src/views/DictationShortcutsPanel.tsx` 负责识别模型、全局快捷键、声音来源、注入方式。
- `ui/src/views/RealtimeSubtitlesPanel.tsx` 负责实时字幕设置、启动/停止、快捷键、声音来源、字幕样式、预览和颜色字段。
- `ui/src/features/dictation/controller.ts` 提供听写启动、快捷键捕获、注入方式设置等逻辑。
- `ui/src/features/subtitles/controller.ts` 提供字幕启动、快捷键捕获、预览窗口同步等逻辑。
- `ui/src/features/audio/devices.ts` 提供音频设备列表。
- 这两个页面当前布局较松散，分组层级和表单对齐不如参考图稳定。

## 实施方案

使用 1.2 中统一后的页面和表单组件重排页面。语音输入页按“识别设置”“输入行为”分组；实时字幕页按“基础设置”“字幕样式”分组。保留所有现有事件处理函数、store 读取和 patch 调用。

## 执行步骤

1. 将 `DictationView.tsx` 页头替换为统一 `PageHeader`，保留 tabs 和 debug 页签逻辑。
2. 将 `DictationShortcutsPanel.tsx` 改为参考图式分组布局：识别模型/声音来源、全局快捷键/注入方式，说明文字放在输入行为分组下。
3. 保留 `startShortcutCapture`、`setInjectMethod`、`patchDictPrefs`、`useAudioDevices` 的调用方式不变。
4. 将 `RealtimeSubtitlesPanel.tsx` 页头改为左侧标题描述状态、右侧开始/停止按钮。
5. 将实时字幕基础设置重排为两列表单，保留模型、快捷键、声音来源、更新方式、字体、位置。
6. 将字幕样式重排为参考图式预览开关条、滑块网格、颜色字段；保留 `syncSubtitleIndicator`、`showSubtitlePreview`、`hideSubtitlePreview` 生命周期。
7. 对无法自动验证的交互写清人工验证步骤：快捷键修改、字幕开始/停止、预览开关、滑块、颜色选择、Select 搜索。
8. 运行 `npx tsc --noEmit` 和 `npm run ui:build`。
9. 更新本任务执行记录和 `00-任务总览.md`。

## 涉及内容
