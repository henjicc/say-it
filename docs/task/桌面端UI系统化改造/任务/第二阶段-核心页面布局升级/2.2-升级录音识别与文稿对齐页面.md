# 2.2 升级录音识别与文稿对齐页面

## 任务状态

- 状态：待开始
- 所属阶段：第二阶段-核心页面布局升级
- 前置任务：1.2
- 关联任务：4.1、4.2
- 是否可以并行：可以。可与 2.1、3.1、3.2 在 1.2 完成后并行，但需避免同时修改 `filePicker.tsx`。

## 任务目标

按参考图升级录音识别和文稿对齐页面，统一上传区、状态条、结果展示、参数设置和文稿输入布局，同时保留现有文件选择、拖放、识别、取消、导出和对齐逻辑。

## 当前情况

- `ui/src/views/TranscriptionView.tsx` 包含录音识别页头、tabs、录音转写、通用设置和结果展示。
- `ui/src/views/TranscriptAlignPanel.tsx` 包含文稿对齐的文件选择、文稿输入、对齐运行、结果导出。
- `ui/src/features/transcription/filePicker.tsx` 封装文件选择、拖放、格式校验和 `FileDropSection`。
- `ui/src/features/transcription/controller.ts` 提供识别、取消、打开设置、对齐等业务动作。
- `ui/src/features/transcription/subtitles.ts` 提供字幕结果转换、SRT 导出相关工具。
- 当前上传区已有基础样式，但页面整体分组、状态条、结果区与参考图不完全统一。

## 实施方案

先统一 `FileDropSection` 的上传区样式，再调整 `TranscriptionView` 和 `TranscriptAlignPanel` 的页面骨架。录音转写页按参考图保留大上传区、开始按钮和状态条；文稿对齐页沿用同一上传区与状态条，并优化文稿输入和结果表格的视觉层级。

## 执行步骤

1. 调整 `FileDropSection`：上传图标、虚线边框、hover、drag active、已选文件信息、错误/警告提示统一使用 token。
2. 调整 `TranscriptionView.tsx` 页头、tabs、录音转写 section、开始/取消/API Key 按钮、状态条。
3. 调整录音识别结果区：纯文本 textarea、字幕列表、复制和导出按钮保持功能不变，视觉与新组件体系一致。
4. 调整通用设置页签：识别模型、语种提示、说话人分离、说话人数使用统一表单布局。
5. 调整 `TranscriptAlignPanel.tsx`：文稿输入区、执行按钮、状态条、两种结果视图、低匹配标注视觉统一。
6. 保留 `useFileDrop`、`useFilePick`、`startTranscription`、`cancelTranscription`、`startAlignment`、`cancelAlignment` 等调用路径不变。
7. 记录人工验证步骤：文件选择、文件拖放、重新选择、开始按钮禁用态、结果导出。
8. 运行 `npx tsc --noEmit` 和 `npm run ui:build`。
9. 更新本任务执行记录和 `00-任务总览.md`。
