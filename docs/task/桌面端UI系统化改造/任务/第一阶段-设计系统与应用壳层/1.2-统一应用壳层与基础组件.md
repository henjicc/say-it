# 1.2 统一应用壳层与基础组件

## 任务状态

- 状态：待开始
- 所属阶段：第一阶段-设计系统与应用壳层
- 前置任务：1.1
- 关联任务：2.1、2.2、3.1、3.2、4.1
- 是否可以并行：否。本任务完成后，页面改造任务才可以并行。

## 任务目标

统一主窗口应用壳层、标题栏、侧栏、主内容容器和基础 UI 组件，为后续页面改造提供稳定的桌面端布局和组件体系。

## 当前情况

- `ui/src/App.tsx` 当前结构为 `Titlebar + Sidebar + main`，主内容最大宽度为 `max-w-5xl`，边距为 `px-8 py-6`。
- `ui/src/components/shell/Titlebar.tsx` 实现 Tauri 无边框窗口控制。
- `ui/src/components/shell/Sidebar.tsx` 实现品牌区和主导航，目前只有语音输入、实时字幕、录音识别、设置。
- `ui/src/components/shell/navConfig.tsx` 使用项目内 SVG 图标。
- `ui/src/components/ui/` 已有 Button、Card、Tabs、Input、Select、Slider、Field、Collapse 等组件，但样式规格不够统一。
- 参考图中的侧栏更宽、导航项更高、主内容区域更像桌面设置面板，页头、分组标题和表单控件层级更稳定。

## 实施方案

在不改变路由和业务状态的前提下，调整应用壳层尺寸与视觉层级，并扩展基础组件能力。新增少量布局组件时放在 `ui/src/components/ui` 或合适的 shell/ui 目录，保持导入路径清晰。

## 执行步骤

1. 调整 `App.tsx` 主窗口布局：主背景层、内容最大宽度、滚动容器、页面边距和低高度窗口表现。
2. 调整 `Titlebar.tsx`：保持 Tauri window API 不变，统一标题栏高度、拖拽区域、窗口按钮 hover/active/focus 状态。
3. 调整 `Sidebar.tsx` 和 `navConfig.tsx`：统一品牌区、导航项尺寸、选中态、hover、图标尺寸；不新增无真实功能的底部入口。
4. 统一 `Button`、`Tabs`、`Input`、`Select`、`Textarea`、`Slider`、`Field`、`Collapse`、`Card` 的视觉规格，使用 1.1 的 token。
5. 新增可复用布局组件：`PageHeader`、`SectionHeader`、`SettingsSection`、`FormGrid`、`StatusBar`、`Switch` 中确有需要的部分。
6. 确认现有页面导入不被破坏，保留基础组件主要 props 和导出名称。
7. 运行 `npx tsc --noEmit` 和 `npm run ui:build`。
8. 更新本任务执行记录和 `00-任务总览.md`。

## 涉及内容

### 新增文件

- `ui/src/components/ui/PageHeader.tsx`：页面标题和描述布局，是否新增按实现时确认。
- `ui/src/components/ui/SectionHeader.tsx`：带蓝色竖线和分割线的分组标题，是否新增按实现时确认。
- `ui/src/components/ui/SettingsSection.tsx`：设置区块布局，是否新增按实现时确认。
- `ui/src/components/ui/Switch.tsx`：统一开关控件，是否新增按实现时确认。

### 修改文件

- `ui/src/App.tsx`：应用壳层布局与主内容容器。
- `ui/src/components/shell/Titlebar.tsx`：窗口控制视觉状态。
- `ui/src/components/shell/Sidebar.tsx`：侧栏布局与导航样式。
- `ui/src/components/shell/navConfig.tsx`：必要时调整图标。
- `ui/src/components/ui/*.tsx`：基础组件视觉统一。

### 可能废弃

- `Card` 当前仅作为分割区块的语义可能需要调整；是否保留或转为新 section 组件，执行时根据页面迁移成本确认。
