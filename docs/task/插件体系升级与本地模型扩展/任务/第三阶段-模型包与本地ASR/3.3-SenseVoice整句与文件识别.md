# 3.3 SenseVoice 整句与文件识别

## 任务状态

- 状态：待开始
- 所属阶段：第三阶段-模型包与本地ASR
- 前置任务：3.2（sherpa-onnx 引擎已接入）
- 关联任务：无
- 是否可以并行：与第四阶段可并行

## 任务目标

基于 3.2 的引擎接入，支持 sherpa-onnx **离线（非流式）**协议族：安装官方 SenseVoice-Small INT8 模型包后，(a) 听写可用"整句模式"（VAD 端点后整句识别，准实时）；(b) 录音识别/听写文件可用本地文件识别。SenseVoice 定位为"可切换的整句高质量模式"，与 Paraformer 实时模式互补。

## 当前情况

- 3.2 完成后已有：`providers/local_asr/` 引擎封装、`local-sherpa-online` 协议族、模型包消费通路。
- sherpa-onnx 提供 `OfflineSenseVoiceModelConfig` 与 Silero VAD 支持（VAD 模型很小，约 2MB，可作为模型包文件之一或内置资源，执行时定并记录）。
- 文件识别通路：`providers/capabilities.rs::FileRecognitionProvider`（`recognize_short` 同步路径 / 异步 OSS 路径），模型 `category: "file"`、`scenes` 含 `dictationFile` / `transcription`；音频解码已有 `symphonia`（mp3/aac/mp4/ogg）可转 PCM。
- 实时"模拟流式"（VAD 分段 + 整句识别）在会话形状上与 3.2 的 `local_session.rs` 相同（channel 进、事件出），只是没有逐字 partial，端点后一次性 final。

## 实施方案

1. 引擎封装扩展离线识别：`OfflineRecognizer` + SenseVoice 配置 + Silero VAD 集成。
2. 新协议族 `local-sherpa-offline`：
   - 实时场景（听写实时/字幕）：`local_session.rs` 增加离线分支——VAD 累积语音段 → 端点触发整句识别 → 发 final（无 partial 或以"识别中"占位，执行时按现有事件语义定）。
   - 文件场景：`FileRecognitionProvider` 新增 `Local` 变体（或经 profile kind 分流，与现有枚举 facade 风格一致），`recognize_short` 走本地：symphonia 解码 → 重采样 → 分段识别 → 拼接结果。
3. 官方 SenseVoice-Small INT8 模型包（双形态），`models` 声明两条场景（realtime + file）或一条多场景，按注册表现有语义定。
4. 长音频防护：文件识别按 VAD 分段循环处理，避免一次性喂超长音频撑内存。

## 执行步骤

1. 扩展 `providers/local_asr/`：离线识别 + VAD；单元测试用短 wav 样本验证输出非空与分段逻辑。
2. `local_session.rs` 离线分支（VAD 端点 → 整句 final）。
3. 文件识别接入 `capabilities.rs`（本地变体）与转写流程分流点。
4. 制作官方 SenseVoice 模型包。
5. 验证：`cargo test`、`npx tsc --noEmit`、`npm run ui:build`；`tauri:dev` 下用一段本地录音文件跑通文件识别并核对文本。
6. 用户实测：整句模式听写一句话（停顿后上屏）；录音识别一个 mp3 文件。
