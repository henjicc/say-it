# 3.2 sherpa-onnx 引擎与本地实时 ASR

## 任务状态

- 状态：待开始
- 所属阶段：第三阶段-模型包与本地ASR
- 前置任务：3.1（模型包机制）
- 关联任务：3.3（复用引擎接入）、2.3（本地供应商出现在 ASR 分类）
- 是否可以并行：否（阶段内串行）

## 任务目标

接入 `sherpa-onnx` Rust crate（CPU、静态链接），实现本地实时 ASR 协议族：安装官方 Paraformer Online INT8 模型包后，听写实时与实时字幕可选中该模型并在断网状态下正常识别（持续 partial + 端点 final）。产出首个官方模型包（内嵌 + 清单双形态）。

## 当前情况

- 实时 ASR 架构：模型注册表 `shared/asr-models.json` 按 `protocol` 字段路由到 `RealtimeAsrConnector` 实现（`providers/alibabacloud/realtime.rs`，trait 含 `connect_request/start_messages/audio_message/finish_message/parse_message`，面向 WebSocket 消息）；会话状态机 `commands/asr/session.rs` 协议无关，只消费 `AsrEvent`。插件实时会话走独立路径 `commands/asr/plugin_session.rs`（音频经 mpsc channel 喂给运行时，事件回发 `emit_asr_stream_event`）。
- **注意**：现有 `RealtimeAsrConnector` trait 是 WebSocket 消息形状的，本地引擎没有"连接请求/消息解析"概念。插件会话路径（channel 进、事件出）与本地引擎形状更接近，更适合作为参照。
- 音频侧：`cpal` 采集、`StreamDsp`（`audio_dsp.rs`）已有重采样/降噪管线，输入采样率在会话启动时传入。
- 新增模型/协议族流程：`docs/rules/新增供应商与模型操作手册.md` 场景二/三。
- 依赖现状：`Cargo.toml` 无 ONNX 相关依赖；已有 vendor 补丁先例（`[patch.crates-io] ocr-rs = { path = "vendor/ocr-rs" }`）应对上游 crate 问题。
- 调研结论（2026-07，见任务背景）：`sherpa-onnx` crate 1.13.4 提供 `OnlineRecognizer` / `OnlineParaformerModelConfig` 安全封装，默认自动下载并静态链接平台原生库；Online Paraformer 中英双语流式 INT8 模型约 250MB，CPU 单路实时余量充足；在线模型不提供时间戳。

## 实施方案

1. **先做链接 PoC**（半天盒装）：加依赖 → `cargo check` → 命令行喂一段 wav 出文字。验证 Windows 构建、产物体积增量、`build.rs` 下载行为对 CI（`.github/workflows/release.yml`）的影响。**PoC 失败即停**，评估 `SHERPA_ONNX_ENABLE_GPU=OFF` 自编译 + `SHERPA_ONNX_LIB_DIR`，结论写入重要记录后再继续。
2. **本地引擎模块**：新增 `providers/local_asr/`（引擎生命周期、模型包定位、`OnlineRecognizer` 封装）。引擎按会话创建、会话结束释放（与 PP-OCR 的按任务构建先例一致，先实测冷启动耗时，若过高再改常驻，结论记录）。
3. **会话接入**：`commands/asr/` 新增 `local_session.rs`（参照 `plugin_session.rs` 的 channel 结构）：PCM 输入 → 重采样 16k 单声道 → `accept_waveform` → 轮询 partial → 端点检测出 final，事件与现有 `emit_asr_stream_event` 完全同构，前端零改动。
4. **注册与路由**：模型来自模型包 manifest 的 `models` 条目，`protocol` 填 `local-sherpa-online`；会话启动处按 protocol 分流到本地会话（现有分流点在 `commands/asr/mod.rs` 或 `application/transcription.rs`，执行时确认写入执行记录）。模型未下载就绪时启动报结构化错误（提示去插件管理下载）。
5. **官方模型包**：用 3.1 构建脚本产出 `sherpa-onnx 中英 Paraformer Online INT8` 双形态包；模型源文件取自 k2-fsa 官方发布，下载 URL 待确认（GitHub Releases 直链国内可达性差，是否转存自有 Release，标记**待确认**）。
6. `supportsVocabulary: false`、`supportsAlignmentTimestamps: false`（在线模型无时间戳）；`scenes` 含 `dictationRealtime` 与 `subtitles`。

## 执行步骤

1. PoC：依赖 + 最小识别验证 + 体积/CI 影响评估，结论写执行记录（与重要记录）。
2. 实现 `providers/local_asr/` 引擎封装 + 单元测试（模型缺失、文件校验失败的错误路径）。
3. 实现 `local_session.rs` 并接入会话分流。
4. 制作官方 Paraformer Online 模型包（双形态）。
5. 验证：`cargo check && cargo test`、`npx tsc --noEmit`、`npm run ui:build`；`tauri:dev` 安装模型包后模型出现在听写实时/字幕下拉。
6. 用户实测（按 CLAUDE.md，麦克风类验证交用户）：断网 → 选中本地模型 → 听写一句话确认注入；开启实时字幕确认滚动；partial 刷新延迟主观可接受。
