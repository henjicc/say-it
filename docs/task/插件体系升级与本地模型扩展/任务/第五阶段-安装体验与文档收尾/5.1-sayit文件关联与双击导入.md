# 5.1 .sayit 文件关联与双击导入

## 任务状态

- 状态：已完成（安装版真实双击与弹窗交互待用户人工回归）
- 所属阶段：第五阶段-安装体验与文档收尾
- 前置任务：3.1（导入确认流程需覆盖模型包形态）
- 关联任务：无
- 是否可以并行：与第四阶段可并行

## 任务目标

Windows 安装时注册 `.sayit` 文件关联；双击 `.sayit` 文件可唤起软件（未运行则启动、已运行则聚焦）并进入导入确认流程（显示插件类型/名称/版本/签名状态，确认后安装），与软件内文件对话框安装同一后端流程。

## 当前情况

- `src-tauri/tauri.conf.json`：无 `fileAssociations` 配置。Tauri 2 支持 `bundle.fileAssociations`（`ext`/`name`/`description`/`mimeType`），NSIS 打包时写注册表。
- 已装 `tauri-plugin-single-instance`（`Cargo.toml`），二次启动的命令行参数会回调到首实例——双击文件在"已运行"场景下由此获得文件路径；"未运行"场景从首实例启动参数（`std::env::args`）获取。
- 现有安装流程：`plugin_management.rs::install_provider_plugin(source_path, allow_unsigned, trust_signing_key)`；前端 `PluginManagerPanel.tsx` 已有未签名/信任确认交互，可复用为导入确认弹窗的主体。
- 主窗口初始 `visible: false`、窗口显示逻辑在后端（`application/window_lifecycle.rs`）——唤起时需确保主窗口显示并导航到插件管理位置。
- 开发环境注意：文件关联仅在安装版生效，`tauri:dev` 下用命令行参数模拟验证。

## 实施方案

1. `tauri.conf.json` 加 `bundle.fileAssociations`：`ext: ["sayit"]`，名称/描述用中文（如「说吧！插件包」）。
2. 启动参数处理：启动时扫描 `env::args` 中存在的 `.sayit` 路径；single-instance 回调中同样解析二次启动参数。两路汇合到同一个"待导入队列"，发前端事件。
3. 前端导入确认：收到事件 → 显示主窗口 → 弹出导入确认（读取包内 manifest 预览：类型徽标[在线插件/模型包]、名称、版本、能力、签名状态；需要新的"预览包"命令，只解包读 manifest 不安装）→ 确认后走现有 `install_provider_plugin`（含未签名/信任分支）→ 完成后跳转插件管理页。
4. 安全注意：路径来自外部输入，后端校验存在性与 `.sayit` 后缀；预览解析失败给出明确错误而非崩溃；确认弹窗必须默认取消（防误装）。

## 执行步骤

1. `tauri.conf.json` 文件关联配置。
2. 后端：启动参数与 single-instance 参数解析、"预览包 manifest"命令、待导入事件。
3. 前端：导入确认弹窗（复用 `Modal` 组件与 PluginManagerPanel 的信任交互）、事件监听（注意仅主窗口入口处理，indicator 窗口不处理）。
4. 验证：`tauri:dev` 下以命令行参数模拟两种场景（未运行启动带参数；已运行时再启动一次带参数）；`npm run tauri:build` 出安装包后实测真实双击（含在线插件包与模型包各一次）。
