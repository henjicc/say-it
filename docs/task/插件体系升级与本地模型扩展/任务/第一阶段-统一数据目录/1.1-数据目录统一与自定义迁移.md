# 1.1 数据目录统一与自定义迁移

## 任务状态

- 状态：已完成（待用户实机验证迁移流程）
- 所属阶段：第一阶段-统一数据目录
- 前置任务：无
- 关联任务：3.1（models 目录布局）、4.1（PP-OCR 模型落位）
- 是否可以并行：可与 2.1、2.2 并行

## 任务目标

所有应用数据收拢到单一数据根目录（默认 `AppData\Local\com.henjicc.sayit`），用户可在设置中改到任意位置并整目录迁移；AppData 原位置只留指针文件。迁移后各功能模块自动使用新路径，无法热刷新的部分提示或自动重启。

## 当前情况

- 设置文件：`src-tauri/src/persistence.rs`，`say-it-state.json` 位于 `app.path().app_local_data_dir()`（`state_file_path`）；已有旧标识符迁移先例（`LEGACY_APP_IDENTIFIERS`，从 `com.vibecode.sayit` 搬状态文件）。
- 插件目录：`src-tauri/src/providers/plugin.rs::plugins_dir()` = `app_local_data_dir()/plugins`。
- OCR 模型：安装目录 `resources/ocr`（本任务不动它，4.1 负责搬进数据目录；但本任务要预留 `models/` 子目录约定）。
- 路径获取分散在各模块直接调用 `app.path().app_local_data_dir()`；`active_app_context/mod.rs:327` 在启动时用 `configure_model_root` 注入 OCR 模型路径。
- 密钥不受影响：插件会话密文在系统 keyring（`plugin_secrets.rs`），不在文件里。

## 实施方案

新增一个数据根目录解析模块（建议 `src-tauri/src/application/data_root.rs` 或挂在 `persistence.rs` 旁）：

1. **启动引导**：读 `app_local_data_dir()/data-root.json` 指针文件（内容形如 `{ "root": "D:\\SayItData" }`）；不存在或指向无效路径则用默认目录（即 `app_local_data_dir()` 本身）。解析结果存入 `OnceLock`/`RuntimeState`，全程序唯一入口函数（如 `data_root()` / `data_subdir("plugins")`）。
2. **收拢调用点**：`state_file_path`、`plugins_dir` 及后续 `models/` 全部改为经该入口取路径；grep `app_local_data_dir` 确认无遗漏（`plugin_package.rs` 的 trusted keys 存储也要检查）。
3. **迁移命令**（Tauri command）：选目标目录 → 校验（非原目录子目录、可写、剩余空间 ≥ 现有数据大小）→ 整目录复制（逐文件，产生进度事件）→ 复制后校验（文件数与字节数比对）→ 写指针文件 → 删除旧目录数据（指针文件所在的 AppData 根目录本身保留）→ 通知前端。
4. **刷新策略**：路径统一从入口函数动态获取的模块自动生效；`configure_model_root` 是 `OnceLock` 一次性注入，迁移后无法热更新 → 迁移完成弹窗提示重启（或调用 `app.restart()`），首版选提示重启即可。
5. **设置 UI**：在 `SettingsView` 合适的标签页（建议「启动与常规」`SettingsStartupPanel.tsx` 或新增小节）加"数据目录"设置：显示当前路径、「更改位置」按钮（目录选择对话框）、迁移进度与结果提示。

失败回退：复制或校验失败 → 不写指针、不删旧目录、清理半成品目标目录，报错并保持原状。

## 执行步骤

## 执行记录

### 2026-07-18

- 新增 `src-tauri/src/application/data_root.rs`：指针文件（默认目录下 `data-root.json`）解析、`data_root()/data_subdir()/data_file()` 统一入口（`OnceLock` 启动解析一次）、迁移命令与 9 个单元测试。
- 迁移采用**白名单**而非整目录复制：`say-it-state.json(+.bak)`、`trusted-plugin-keys.json`、`plugins/`、`plugin-data/`、`plugin-webviews/`、`cues/`、`models/`（为 3.1 预留）。原因：默认目录下的 `EBWebView`（WebView2 运行时缓存）被进程独占且无法重定位，指针文件必须留在原位——因此迁移后默认目录会剩指针文件 + `EBWebView`，与计划中"仅剩指针文件"略有出入，属必要偏差。
- 迁移流程：目标校验（绝对路径、非当前目录内部、空目录/迁回默认时无数据条目冲突、可写探针、Windows `GetDiskFreeSpaceExW` 剩余空间 ≥ 数据量 + 64MiB）→ 逐文件复制（`data-root-migration` 进度事件，100ms 节流）→ 逐文件字节数校验 → 写/删指针 → 删除旧数据白名单条目。失败时清理目标端半成品、不写指针、旧数据保持原状。
- 收拢调用点：`persistence.rs::state_file_path`、`plugin.rs::plugins_dir`、`plugin_package.rs`（trusted keys 读写、plugin-data/plugin-webviews 卸载清理）、`plugin_management.rs`（登录 WebView data_directory 与重置）、`settings.rs`（cues 提示音目录）。`plugin.rs:722` 的 plugin-data 由插件根目录反推，随 plugins 目录自动生效，无需改动。保留不动并记录原因：`persistence.rs::legacy_state_file_paths`（旧标识符迁移，固定 AppData）、`main.rs` 的 OCR 模型 `resources/ocr`（4.1 处理）、窗口/资源类路径（非数据）。
- 新命令：`get_data_root_status`（activeRoot/configuredRoot/defaultRoot/isCustom/restartRequired）、`migrate_data_root`（`spawn_blocking` 执行 + `AtomicBool` 防并发）、`restart_app`。
- 前端：「设置 → 启动设置」新增「数据目录」小节（当前路径、更改位置、恢复默认位置、迁移进度条、完成后「立即重启」按钮）。
