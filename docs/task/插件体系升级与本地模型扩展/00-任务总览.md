# 任务总览：插件体系升级与本地模型扩展

## 任务目标

「说吧！」当前的 `.sayit` 插件体系只能接入在线 ASR/翻译 API（QuickJS 沙箱连接器），本地模型（PP-OCR）以硬编码内置方式存在，设置页「密钥与识别」只有单一的「识别供应商」分类。本任务要把插件体系升级为覆盖 **ASR / OCR / 翻译** 三类能力的统一扩展机制，并新增**模型包**插件形态（`.sayit` 包内携带本地模型文件，无代码），使本地语音识别模型（sherpa-onnx 系：Paraformer Online、SenseVoice）能按需安装、PP-OCR 从内置改为模型包，同时：

- 所有应用数据（设置、插件、模型）统一到一个可自定义位置的数据目录，支持全量迁移；
- `.sayit` 文件通过安装时注册的文件关联实现双击导入；
- 设置页按「ASR 供应商 / OCR 供应商 / 翻译供应商」重新分类。

**属于本次任务**：数据目录统一与迁移、OCR 能力类型接入供应商框架、插件回环网络权限、设置页重构、模型包格式与安装、sherpa-onnx 本地 ASR（Paraformer Online + SenseVoice）、PP-OCR 模型包化、场景感知 OCR 供应商选择、`.sayit` 文件关联、操作手册与技能同步。

**不属于本次任务**：原生代码插件（DLL）、sidecar 进程插件、GPU 加速（首版仅 CPU INT8）、独立的图片转文字功能 UI（仅预留接口）、macOS 打包适配、Fun-ASR-Nano / Qwen3-ASR 本地化（后续按模型包机制自然扩展）。

## 重要说明

- **插件 API 版本兼容**：当前 `PLUGIN_API_VERSION = 3` 且校验为精确匹配（`plugin.rs::validate_manifest`）。本次新增 ocr 能力与 model-pack 形态时升到 v4，但宿主必须**同时接受 v3 与 v4**，已发布的 v3 在线插件不得失效。
- **本地模型只走「内置引擎 + 模型包」路线**：`.sayit` 包禁止携带可执行文件/动态库的校验（`plugin_package.rs`）不放松；模型包只含模型权重与清单。
- **决策依据见** [重要记录.md](重要记录.md)：OCR 接口按通用设计（预留独立功能）；模型包以内嵌模型为主、同步支持清单下载包；数据目录全量迁移、AppData 留指针文件。
- 每个任务完成后独立提交 Git，遵循仓库 CLAUDE.md 的提交约定。
