# 供应商插件化 · 任务总览

## 目标

让用户在不下载或修改「说吧！」源码的前提下，通过独立插件包增加语音模型和供应商；同时提供 Skill，引导 AI 调研供应商、生成连接器、校验清单并完成安装。

## 架构边界

- Rust 后端是模型目录、供应商配置、音频处理、会话状态和插件进程生命周期的唯一权威。
- 插件以独立进程运行，通过带版本的 JSONL 协议接收处理后的 PCM 并返回统一识别事件。
- 前端只渲染后端提供的模型、供应商与配置字段，不包含供应商协议分支。
- 网页登录、Cookie 和逆向接口属于特权插件；不得注入主窗口或悬浮窗，也不得读取未授权会话。

## 阶段

| 阶段 | 范围 | 状态 |
|---|---|---|
| 1. 插件 API v1 | 外置清单、动态模型/供应商目录、实时 ASR 进程协议、通用配置 UI | 自动验证通过，待真实插件验证 |
| 2. 插件 API v2 | 文件识别、字幕翻译、热词、通用动作和可取消的一次性进程协议 | 自动验证通过，待真实插件验证 |
| 3. 特权宿主 | 独立网页登录 WebView、URL 白名单 Cookie、DPAPI 会话保护、显式权限确认 | 自动验证通过，待真实插件验证 |
| 4. 分发与信任 | SHA-256 完整性、Ed25519 签名、发布者首次信任、原子更新与回滚 | 自动验证通过，待真实插件验证 |

## 宿主架构验收

- 应用启动时扫描 `%LOCALAPPDATA%\com.henjicc.sayit\plugins`，损坏插件不会阻止应用启动。
- 插件模型出现在语音输入和字幕模型列表中，选择模型会自动路由到其供应商。
- 配置字段由清单驱动；密钥只在后端保存，不返回前端快照。
- 主窗口关闭后，插件实时识别仍由后端进程运行。
- 插件崩溃、非法 JSON、认证失败和收尾超时会转换为统一领域错误并回收进程。
- `adapt-sayit-provider` Skill、清单验证器、安装器和连接器模板均通过校验与烟雾测试。
- 文件识别、翻译、热词和自定义动作通过 v2 `invoke` 协议运行；取消文件任务会终止插件进程。
- 网页逆向类供应商只需在清单声明登录页、Cookie URL 白名单与捕获脚本；宿主不包含供应商专属逻辑。
- Cookie 和页面捕获参数不进入 React 或普通配置，使用当前 Windows 用户的 DPAPI 加密保存。
- `.sayit` 是统一单文件 ZIP 分发格式；插件管理器先读取包内 `sayit-package.json`，按 `kind` 分派。当前支持 `provider-plugin`，并会在私有暂存目录解压、完成结构、哈希和签名校验后再安装，旧版本进入备份目录并支持回滚。

## 非目标与剩余人工验证

- 本阶段不实现任何具体新供应商，也不把豆包逆向代码复制进「说吧！」；豆包仅作为后续由 AI 按 Skill 生成插件的架构验收样本。
- 当前支持从本地 `.sayit` 文件手动安装、更新和回滚；开发目录安装仅保留为后端兼容入口，未实现联网插件市场或自动下载更新。
- 网络权限是声明、审查和用户确认契约，并非操作系统级网络沙箱；插件仍按本机用户权限运行，因此分发包应优先使用受信任签名。
- 真实供应商的账号登录、Cookie 有效期、上游协议兼容和识别质量必须由生成出的插件单独验证。
