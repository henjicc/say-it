# 供应商插件化 · 任务总览

## 目标

让用户在不下载或修改「说吧！」源码的前提下，通过平台无关的 JavaScript 插件包增加语音模型和供应商；同时提供 Skill，引导 AI 调研供应商、生成连接器、校验、签名并交付单个 `.sayit` 文件。

## 架构边界

- Rust 后端是模型目录、供应商配置、音频处理、会话状态、网络资源和 JavaScript 运行时生命周期的唯一权威。
- 插件是由内嵌 QuickJS 执行的 API v3 JavaScript 模块；音频以 `Uint8Array` 传入，不启动外部进程。
- 前端只渲染后端提供的模型、供应商与配置字段，不包含供应商协议分支。
- 网页登录、Cookie 和逆向接口属于特权插件；不得注入主窗口或悬浮窗，也不得读取未授权会话。

## 阶段

| 阶段 | 范围 | 状态 |
|---|---|---|
| 1. 插件 API v1/v2 | 历史外部进程与 JSONL 协议 | 已停用；安装时明确提示重新生成 |
| 2. 插件 API v3 | QuickJS、受控宿主 API、实时/文件/翻译统一能力 | 自动验证通过，待真实插件验证 |
| 3. 特权宿主 | 独立网页登录 WebView、URL 白名单 Cookie、系统凭据库存储、显式权限确认 | 自动验证通过，待真实插件验证 |
| 4. 分发与信任 | 纯源码包校验、SHA-256、Ed25519、首次信任、原子更新与回滚 | 自动验证通过，待真实插件验证 |

## 宿主架构验收

- 应用启动时扫描 `%LOCALAPPDATA%\com.henjicc.sayit\plugins`，损坏插件不会阻止应用启动。
- 插件模型出现在语音输入和字幕模型列表中，选择模型会自动路由到其供应商。
- 配置字段由清单驱动；密钥只在后端保存，不返回前端快照。
- 主窗口关闭后，插件实时识别仍由 Rust 后端的隔离 QuickJS 上下文运行。
- JavaScript 异常、越权网络、非法导入、认证失败、取消和收尾超时会转换为统一领域错误并关闭网络与运行时资源。
- `adapt-sayit-provider` Skill、清单验证器、签名/归档脚本和 JavaScript 模板均通过校验与烟雾测试。
- 文件识别、翻译、热词和自定义动作通过 v3 `invoke` 运行；文件只暴露不透明输入句柄。
- 网页逆向类供应商只需在清单声明登录页、Cookie URL 白名单与捕获脚本；宿主不包含供应商专属逻辑。
- Cookie 和页面捕获参数不进入 React 或普通配置；Windows 从旧 DPAPI 文件迁移到系统凭据存储，macOS 使用 Keychain。
- `.sayit` 是统一单文件 ZIP 分发格式；插件管理器先读取包内 `sayit-package.json`，按 `kind` 分派。当前支持 `provider-plugin`，并会在私有暂存目录解压、完成结构、哈希和签名校验后再安装，旧版本进入备份目录并支持回滚。
- 包内只允许 JSON、JavaScript 与必要资源；PE、ELF、Mach-O、动态库、原生模块和符号链接会被拒绝。

## 非目标与剩余人工验证

- 本阶段不实现任何具体新供应商，也不把豆包逆向代码复制进「说吧！」；豆包仅作为后续由 AI 按 Skill 生成插件的架构验收样本。
- 当前支持从本地 `.sayit` 文件手动安装、更新和回滚；开发目录安装仅保留为后端兼容入口，未实现联网插件市场或自动下载更新。
- 插件没有 Node、DOM、文件系统、环境变量、进程、Shell、Tauri IPC 或原生模块能力；网络请求和重定向由宿主按 HTTPS/WSS 主机白名单执行。
- 真实供应商的账号登录、Cookie 有效期、上游协议兼容和识别质量必须由生成出的插件单独验证。
