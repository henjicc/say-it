# 2.2 补齐供应商能力接口与通用服务

## 任务状态

- 状态：待开始
- 所属阶段：第二阶段-供应商能力后端化
- 前置任务：2.1
- 关联任务：3.1、3.2、4.1、4.2
- 是否可以并行：否；它是后续业务迁移的供应商边界

## 任务目标

在现有实时识别连接器基础上，为文件识别、文本翻译和热词/词表等真实能力建立供应商边界，并让上层业务只调用通用应用服务。以后接入其他供应商时，不需要修改听写、字幕、转写或对比状态机。

## 当前情况

- `providers/connector.rs` 已定义 `RealtimeAsrConnector`，实时会话循环基本与阿里云协议解耦。
- 文件识别仍直接依赖 `providers/alibabacloud/transcription/`；`commands/transcription.rs` 负责选择同步/异步通路。
- 翻译命令直接调用 `providers/alibabacloud/translation.rs`。
- 热词命令和前端 API 名含 `funasr_*`，属于供应商专属入口。
- `providers/mod.rs::realtime_connector_for` 已按 `kind` 分发，但没有统一的文件、翻译和 customization 工厂。
- 当前只有一个供应商，抽象必须服务于已存在的真实差异，不能设计成通用插件平台。

## 实施方案

- 保留实时连接器的低层消息职责，应用服务负责会话和重连。
- 为文件识别定义能够表达“提交、进度、取消、结果”的能力接口；供应商内部继续自行决定 OSS 异步、SSE 或短音频同步，不把协议细节泄露给业务层。
- 为文本翻译定义流式结果接口，应用层负责请求代次和字幕分句，上游只返回标准化增量/完成/错误事件。
- 热词采用可选能力接口或 capability service；通用命令携带 provider ID，不再把 `funasr` 写进公共命令名。旧命令保留兼容代理到 5.1。
- 为测试提供 fake provider，实现无需真实网络即可覆盖状态机、取消和错误路径。

## 执行步骤

1. 盘点实时、文件、翻译和热词的输入输出、取消方式、错误类型与供应商专属字段，形成最小标准事件。
2. 在 `providers/` 定义文件识别、翻译和可选 customization 能力接口或等价服务边界，明确 `Send/Sync` 和异步返回方式。
3. 让阿里云现有模块实现这些边界，逐段搬运，不改现有协议、上传、解析和错误文案语义。
4. 新增按 provider profile/kind 获取能力实现的工厂；能力缺失返回结构化错误，而不是在业务层判断供应商名。
5. 将 `commands/transcription.rs`、`commands/translation.rs`、`commands/funasr.rs` 改为调用通用服务；旧命令保留薄兼容层。
6. 增加 fake provider 和单元测试，覆盖同步完成、异步进度、取消、迟到事件、翻译流错误、能力不支持和热词操作失败。
7. 按《新增供应商与模型操作手册》模拟登记一个不访问网络的测试供应商，确认业务层无需修改后移除测试数据。

## 涉及内容

### 新增文件

- `src-tauri/src/providers/file_recognition.rs`、`translation.rs`、`customization.rs` 或执行时确定的等价能力模块。
- `src-tauri/src/providers/testing.rs`：仅测试使用的 fake provider。
- 必要的应用服务 facade，预计位于 `src-tauri/src/application/providers.rs`。

### 修改文件

- `src-tauri/src/providers/mod.rs`、`connector.rs`：统一能力发现和工厂。
- `providers/alibabacloud/transcription/`、`translation.rs`、`customization.rs`：实现通用能力，不改协议行为。
- `src-tauri/src/commands/transcription.rs`、`translation.rs`、`funasr.rs`：变为薄传输层和兼容代理。
- `ui/src/lib/tauri.ts`、供应商设置/热词页面：逐步切换通用命令。
- `docs/rules/新增供应商与模型操作手册.md`：记录真实接口和新增步骤。

### 可能废弃

- 公共层的 `funasrSaveHotwords`、`funasrSyncHotwords`、`funasrClearHotwords` 名称：5.1 删除，阿里云 provider 内部命名不受此限制。
- 业务层直接调用 `alibabacloud` 模块的路径。

## 特殊风险

- 不允许把阿里云异步 task ID、OSS URL 等专属字段强塞进所有供应商公共接口；可用 opaque metadata 或 provider 内部状态处理。
- 不为了抽象而统一本质不同的热词机制；能力接口可以返回“不支持”或暴露供应商专属扩展信息。
- trait async 方案优先使用现有语言/依赖能力，新增 `async-trait` 等依赖前必须说明必要性。

## 验收标准

- [ ] 听写、字幕、转写和对比上层代码不需要判断 `alibabacloud-funasr` 或调用阿里云模块。

- [ ] 翻译流的增量、完成、错误和取消均有标准化事件与测试。
- [ ] 热词通用入口按 provider ID 分发，旧命令仍兼容。
- [ ] fake provider 可在无网络环境覆盖成功、失败、取消和迟到事件。
- [ ] 操作手册能指导新增供应商能力实现，不要求修改业务状态机。
- [ ] `cargo check`、相关测试、前端构建通过。
- [ ] 任务总览已同步更新。
