# 1.2 统一配置持久化与状态快照

## 任务状态

- 状态：待开始
- 所属阶段：第一阶段-基线与状态归一
- 前置任务：1.1
- 关联任务：3.1、3.2、3.3、4.1、4.2、5.1
- 是否可以并行：可以；可与 2.1 并行，避免同时修改 `main.rs` 和公共契约文件

## 任务目标

把目前分散在 Rust 状态文件和 WebView `localStorage` 中的持久化配置统一交给 Rust 管理，并让任意新建的主窗口能够通过一个完整快照恢复页面。迁移必须无损兼容现有用户配置，并保留旧版本回退能力。

## 当前情况

- `src-tauri/src/persistence.rs` 的 `PersistedData` 只保存 providers、听写快捷键、字幕快捷键、字幕翻译模型、启动设置和 OBS 配置，写入使用直接 `fs::write`。
- `useDictPrefs.ts` 保存 `sayItDictPrefs`，包含模型、麦克风、DSP、静音断流、本地规则和提示音选择。
- `useSubtitleStore.ts` 保存 `sayItSubtitlePrefs`，包含音源、模型、样式、动画、翻译和 OBS 开关。
- `useCompareStore.ts` 保存 `sayItComparePrefs`；`useThemeStore.ts` 保存 `sayItAccentTheme`。
- `SettingsMicCuePanel.tsx` 还把自定义提示音 Data URL 单独保存在 `localStorage`。
- Rust 无法在没有 WebView 时直接读取这些键，因此销毁主窗口后不能恢复完整配置。

## 实施方案

- 扩展后端配置结构，按领域拆为明确类型，而不是继续向一个无结构 JSON 堆字段。
- `PersistedData` 增加 schema 版本和新领域字段，所有新增字段使用默认值并写旧 JSON 反序列化测试。
- 保存改为“序列化到临时文件 → 刷新 → 替换目标文件”，并保留上一份备份；Windows 替换细节执行时按现有依赖确认。
- 增加一次性导入命令：前端读取旧键，传给 Rust；Rust 只在尚未导入或用户明确重试时合并，不能覆盖后端已有新值。
- 迁移期前端 store 仍保持相同公开 API，但初始化改为拉取后端快照，修改改为调用后端；成功后更新本地缓存并镜像旧 `localStorage`，便于旧版本回退。
- UI 专属的导航、弹窗、焦点状态不持久化到 Rust。主题值持久化到 Rust，但 CSS 变量计算与应用仍在前端。

## 执行步骤

1. 列出全部旧键、默认值、夹取规则和历史迁移逻辑，建立 Rust 对等类型与纯函数测试。
2. 为 `PersistedData` 增加 schema 版本和领域配置，验证空 JSON、当前 JSON、旧 identifier 路径 JSON 均能读取。
3. 实现原子保存和备份恢复；模拟损坏主文件，确认可以给出明确错误或从备份恢复，禁止静默重置全部配置。
4. 实现配置快照、按领域更新命令和一次性旧数据导入命令；更新必须先校验再落盘，失败不能只改内存。
5. 前端新增启动迁移桥：读取五类旧数据，调用导入后重新获取快照；导入成功标记由后端保存。
6. 将 `useDictPrefs`、`useSubtitleStore`、`useCompareStore`、`useThemeStore` 改为后端快照缓存；保留旧键兼容镜像。
7. 处理自定义提示音：把 Data URL 导入应用数据目录并保存元数据；此任务只迁移数据，实际原生播放在 3.1 完成。
8. 验证升级、重复导入、部分旧数据、非法字段、保存失败和旧版本回退场景。

## 涉及内容

### 新增文件

- `src-tauri/src/application/settings.rs` 或执行时确认的同职责模块：领域配置、归一化和更新服务。
- 配置迁移与持久化测试文件。
- 必要的前端一次性迁移桥模块，预计位于 `ui/src/features/settings/`。

### 修改文件

- `src-tauri/src/persistence.rs`：版本化结构、原子写入、备份和兼容读取。
- `src-tauri/src/state.rs`：加入后端权威配置状态。
- `src-tauri/src/commands/dictation.rs`、`commands/subtitles.rs`、`commands/session.rs`：逐步收敛配置读写入口。
- `ui/src/store/useDictPrefs.ts`、`useSubtitleStore.ts`、`useCompareStore.ts`、`useThemeStore.ts`：改为快照缓存和命令代理。
- `ui/src/views/SettingsMicCuePanel.tsx`、`ui/src/lib/cues.ts`：迁移自定义提示音存储来源，播放暂保留兼容路径。
- `ui/src/App.tsx` 或启动 hook：在渲染依赖配置的页面前完成快照加载。

### 可能废弃

- 旧 `localStorage` 直接读写函数：5.1 验收前保留兼容镜像，不在本任务删除。
- 分散的单字段 get/set 命令：待所有消费方切换后在 5.1 统一评估。

## 特殊风险与回退要求

- 不得删除旧键；导入命令必须幂等。
- 自定义规则字段不能因 Rust 默认值覆盖用户排序或自定义表达式。
- 保存失败必须返回错误并保留旧文件；不得出现“内存显示成功、磁盘未保存”。
- 回退旧版本前，前端兼容镜像必须包含用户在新版本界面中修改的设置。

## 验收标准

- [ ] 所有现有持久化配置均有 Rust 类型、默认值和归一化测试。
- [ ] 现有 `say-it-state.json` 与五类旧 `localStorage` 数据可无损迁移。
- [ ] 导入重复执行不会覆盖更新后的后端值或生成重复提示音文件。
- [ ] 主窗口重新创建后，只依赖后端快照即可恢复设置。
- [ ] 保存采用可恢复写入方式，损坏/失败路径有测试和明确错误。
- [ ] 前端 store 不再是持久化权威，但旧版本回退镜像仍可用。
- [ ] 前端构建、`cargo check` 和相关测试通过。
- [ ] 用户核对设置页、主题、字幕样式、本地规则和自定义提示音配置没有丢失。
- [ ] 任务总览已同步更新。
