# 5.1 收敛前端职责并完成总体验收

## 任务状态

- 状态：待验证
- 所属阶段：第五阶段-前端收敛与总体验收
- 前置任务：3.3、4.1、4.2
- 关联任务：全部已完成任务
- 是否可以并行：否；需要在所有领域迁移完成后统一收尾

## 任务目标

删除迁移期兼容运行路径和无消费接口，确认前端最终只保留展示与高频交互职责；完成全功能、持久化、供应商扩展、内存、CPU 和时延验收，并将可维护的最终边界同步到项目规则。

## 当前情况

- 前四阶段会有意保留旧命令、旧事件、`localStorage` 镜像和部分薄适配器，以便逐领域回退。
- 若不集中收尾，项目会长期存在两套概念和死代码，后续 AI 可能误用旧 controller。
- 仓库没有统一 lint/自动化 UI 测试，最终必须结合 Rust 测试、前端构建、进程采样和用户交互验证。
- `docs/rules/新增供应商与模型操作手册.md` 需要反映新的后端能力边界。
- 本计划不要求把纯 UI 状态或高频 Canvas/指针交互迁到 Rust，验收时必须按实施方案的最终边界判断，而不是按“前端行数越少越好”。

## 实施方案

1. 以“运行时唯一权威”和“无主 WebView 可运行”为准清理旧路径，而不是机械删除所有 TypeScript 业务文件。
2. 通过静态搜索建立残留清单：前端 session ID、ASR/转写事件分发、raw PCM、业务 timer、直接 `localStorage`、模型注册表 import、供应商协议判断和 provider 专属公共命令。
3. 删除已完成兼容窗口的命令/事件；保留的数据兼容代码必须注明移除条件和最早版本，不留无期限临时代码。
4. 使用 fake provider 完成“新增测试供应商/模型”演练，证明业务服务无需修改；测试完成后不保留虚假生产数据。
5. 按 1.1 相同机器、构建类型和采样方法进行最终对比，分别测量前台、托盘、听写后和字幕运行。

## 执行步骤

1. 审阅每个任务执行记录和提交，确认不存在“待验证但先清理旧路径”的领域。
2. 使用 `rg` 搜索旧 controller、raw 事件、直接注册表 import、`localStorage` 和 provider 硬编码，逐项标记删除、保留或待确认原因。
3. 删除无消费的 Tauri 命令/事件、TypeScript 状态机和迁移开关；收紧 Rust 可见性与模块依赖。
4. 决定旧 `localStorage` 后续策略：建议至少保留读取兼容一个稳定版本，停止继续写入的时间点待确认；不得在未发布验证前清空用户浏览器数据。
5. 更新 `AGENTS.md` 中项目专属架构边界和 `docs/rules/新增供应商与模型操作手册.md`；仅记录稳定最终规则。
6. 运行前端构建、Rust 检查、相关全量测试和持久化迁移测试；按经验要求先 UI build 再 cargo check。
7. 用 fake provider 演练新增同族模型、新实时供应商、新文件识别能力和翻译能力，确认修改范围符合手册。
8. 由用户按完整清单测试窗口、托盘、听写、字幕、OBS、转写、对齐、字幕编辑、对比、音频调试、设置和升级配置。
9. 执行最终进程树采样并与 1.1 基线比较；记录内存、CPU、进程数、恢复时延和快捷键响应。
10. 若达到验收条件，更新全部任务状态并创建收尾提交；未达标则明确剩余瓶颈，不用工作集裁剪等伪优化掩盖。

- `src-tauri/src/commands/`、`main.rs`、`state.rs`：删除旧命令/事件和迁移开关，保持薄传输层。
- `src-tauri/src/application/`、`providers/`、`desktop/`：收紧最终依赖和公开接口。
- `AGENTS.md`：更新最终架构、运行命令和已知陷阱。
- `docs/rules/新增供应商与模型操作手册.md`：更新最终新增流程。
- 本计划全部总览、任务执行记录和重要记录。

### 可能废弃

- 前四阶段列出的所有旧前端 controller、raw PCM 事件、provider 专属公共命令和运行时兼容开关。
- 旧 `localStorage` 写镜像；删除读取兼容的版本点待确认。
- 只为主 WebView 存在而设计的快捷键转发事件。

## 验收标准

### 架构

- [ ] Rust 是所有持久化配置和后台领域状态唯一权威。
- [ ] 前端只保留页面、表单草稿、视觉状态、Canvas 和高频交互。
- [ ] `commands/` 不保存跨调用状态，`providers/` 不依赖 UI，`desktop/` 不依赖上层页面。
- [ ] 前端不直接导入模型注册表、不判断供应商协议、不编排 ASR/翻译/转写会话。

### 功能

- [ ] 主窗口销毁后听写、按住说话、取消、字幕、翻译、OBS 和后台任务符合契约。
- [ ] 主窗口重建后配置和运行状态正确，无重复任务。
- [ ] 转写、对齐、字幕编辑、模型对比和音频调试无回归。
- [ ] 旧状态文件和 `localStorage` 升级无损，回退路径符合记录。
- [ ] 新供应商演练不修改业务状态机。

### 验证

- [ ] `npm run ui:build` 通过。
- [ ] `cargo check` 通过。
- [ ] 相关 Rust 单元/集成测试和迁移测试全部通过。
- [ ] 用户完成全部鼠标、快捷键、注入、字幕和 OBS 交互测试。
- [ ] 托盘内存达到 1.1 锁定阈值；未达到时已给出真实瓶颈和是否继续优化的结论。
- [ ] 空闲 CPU、主窗口恢复和首次快捷键响应无可感知退化。
- [ ] 任务总览、实施方案、重要记录和项目规则已同步。
