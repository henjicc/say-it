# 3.2 迁移实时字幕翻译与直播输出编排

## 任务状态

- 状态：已完成
- 所属阶段：第三阶段-后台实时核心迁移
- 前置任务：1.2、2.2、3.1
- 关联任务：3.3、5.1
- 是否可以并行：否；需要复用 3.1 的音频会话协调器

## 任务目标

把实时字幕的音源管理、ASR 会话、静音门控、断线重连、分句翻译、滚动/替换文本模型和 OBS 输出路由迁到 Rust。前端只负责字幕设置表单、预览动画和悬浮窗渲染，主 WebView 销毁后真实字幕仍持续工作。

## 当前情况

- `features/subtitles/controller.ts` 约 1000 行，混合了运行时、窗口布局、字幕文案合并、分句翻译、OBS 监控以及设置页预览模拟。
- Rust 已能采集麦克风/系统音频、启动 ASR、发起翻译、管理 OBS 连接和发布 overlay snapshot，但由前端按定时器串联。
- 替换模式有 2.5 秒续接语义、1800 字符上限；滚动模式保留最近行；翻译以 request ID + segment seq 抵御乱序。
- OBS 路由会检测连接状态、强制布局、推送原文/译文快照，相关时序已有经验文档约束。
- 预览模拟属于设置页展示，不应进入后台真实字幕状态机。

## 实施方案

- 建立 Rust `SubtitleService`，以显式状态表示 Idle、WaitingForVoice、Running、Reconnecting、Stopping、Failed。
- 音源只通过 3.1 音频协调器获取，麦克风和系统 loopback 共享一致 owner/cancel 规则。
- 将原文显示模型、替换模式续接、滚动行裁剪、翻译分句和乱序合并实现为不依赖 Tauri 的纯 Rust 模块，并用当前边界值建立测试。
- 翻译通过 2.2 的通用 provider 能力；字幕服务维护 request epoch 与 segment seq。
- OBS 运行时由 Rust 订阅字幕领域状态直接发布，不再由主 WebView `setInterval` 轮询。
- 设置页预览仍可留前端，但使用独立 preview 状态，不写真实运行时、不启动真实音频/网络。
- 悬浮窗继续消费已经成形的原文、译文、布局和动画配置；高频 DOM 动画不迁入 Rust。

## 执行步骤

1. 把当前替换模式、滚动模式、分句切割、翻译乱序和重连策略写成行为测试，覆盖现有经验文档中的历史问题。
2. 扩展音频协调器支持字幕麦克风/系统音频 owner，验证与听写互斥或共享规则与基线一致。
3. 实现字幕状态机和 ASR 生命周期，迁移静音断流、自动重连、最大重试和迟到事件过滤。
4. 实现纯 Rust 字幕文档模型：原文 committed/current、替换续接、行数裁剪和最终显示文本。
5. 迁移翻译分句、请求代次、segment 序号、原译文组合和错误处理。
6. 将 OBS 状态检查、布局同步、快照合并和路由切换改为后端事件驱动；保留连接断开后的现有退化行为。
7. 后端直接更新指示器窗口；前端 controller 改为设置/预览适配和状态订阅。
8. 在主窗口隐藏和临时销毁环境中验证字幕、翻译、OBS 连续运行；本任务仍不切换正式关闭策略。

## 涉及内容

### 新增文件

- `src-tauri/src/application/subtitles/`：状态机、文档模型、翻译编排、OBS 路由和测试。
- 必要的字幕领域 DTO/事件类型，归入 1.1 契约模块。

### 修改文件

- `src-tauri/src/state.rs`：注册字幕服务。
- `desktop/backend_mic.rs`、`backend_system_audio.rs`：通过共享音频协调器提供数据。
- `commands/subtitles.rs`、`commands/translation.rs`、`commands/obs.rs`：改为薄应用服务入口或兼容代理。
- `obs_overlay.rs`：接收字幕服务状态，保持既有快照和网页版本机制。
- `desktop/indicator.rs`：接收后端布局、原文和译文状态。
- `ui/src/features/subtitles/controller.ts`：删除真实运行时编排，保留设置预览相关代码并建议拆文件。
- `ui/src/store/useSubtitleStore.ts`、`hooks/useTauriBridge.ts`：消费快照和领域事件。

### 可能废弃

- 前端真实字幕的 session ID、重连 timer、静音 timer、翻译 Map、OBS monitor 和 snapshot queue。
- `subtitleTranslationEvent` 等只为前端编排设计的事件；5.1 统一删除。
- 麦克风/系统音频 raw chunk 向主窗口广播路径；待 4.2 确认无其他消费方后删除。

## 特殊风险与回退要求

- 替换模式续接、翻译乱序和 OBS 快照都曾出现历史回归，必须先有纯逻辑测试再搬运。
- 设置预览与真实字幕必须完全隔离；关闭预览不能停止正在运行的真实字幕。
- 不允许通过降低字幕刷新频率掩盖 IPC 或锁竞争问题。
