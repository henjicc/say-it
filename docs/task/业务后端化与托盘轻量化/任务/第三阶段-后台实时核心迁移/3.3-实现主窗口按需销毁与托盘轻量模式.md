# 3.3 实现主窗口按需销毁与托盘轻量模式

## 任务状态

- 状态：已完成
- 所属阶段：第三阶段-后台实时核心迁移
- 前置任务：3.1、3.2
- 关联任务：5.1
- 是否可以并行：否；这是后台核心迁移的集成验收

## 任务目标

将点击关闭和静默启动后的主窗口策略从“保留 WebView 并隐藏”升级为“保存状态后销毁主窗口、从托盘按需重建”，并确保听写、字幕、翻译、OBS 和快捷键在无主 WebView 时不受影响。通过统一采样证明内存收益，未达标或影响体验时可以独立回退为 `hide()`。

## 当前情况

- `desktop/window.rs::park_main_window` 当前保存位置、设置跳过任务栏后调用 `hide()`，因此有关闭动画但不会释放主 renderer。
- `main.rs` 在 setup 时创建主窗口、悬浮窗和托盘；主窗口配置 `visible: false`、`decorations: false`。
- `App.tsx` 启动时加载 provider/status 并安装 `useTauriBridge`；窗口重建还没有后端快照握手。
- 悬浮窗启动时常驻一个单独 WebView，React 内容隐藏不等于原生窗口隐藏；是否进一步空闲销毁尚未验证。
- CC Switch 的轻量模式可以销毁主窗口，是因为核心后台功能不依赖前端；3.1/3.2 完成后本项目才具备相同前提。

## 实施方案

### 主窗口生命周期

- 新增明确的主窗口状态：Absent、Creating、Ready、Visible、Closing。
- 关闭请求先记录位置/内容区尺寸和必要 UI 快照，再销毁主窗口；不得触发业务 shutdown。
- 托盘/单实例/显式打开路径统一调用一个幂等 `ensure_main_window()`，从 `tauri.conf.json` 创建窗口并恢复位置尺寸。
- 前端加载后发 `main-window-ready`，后端在 ready 前只排队最新领域状态；前端按“快照 → 订阅 → revision 校正”恢复。

### 指示器策略

- 首先修正原生 `hidden` 状态：空闲时真正 `window.hide()`，显示态 `show()`，解决任务管理器前台归类和无谓合成。
- 第一轮保留指示器 WebView 进程，先测量主窗口销毁收益。
- 若未达到 1.1 阈值，再实现可选的指示器空闲销毁：创建后等待 ready，再发送完整状态；设置足够冷却时间。首次快捷键响应一旦明显变慢则回退，不将其作为硬性完成条件。

## 执行步骤

1. 增加主窗口生命周期状态和幂等创建服务，统一托盘菜单、托盘点击、单实例与启动路径。
2. 实现前端 ready 握手、快照恢复和 revision 校正；模拟创建过程中连续收到多个领域事件，只应用最新状态。
3. 将关闭请求改为销毁主窗口，确认 `beforeunload` 不再停止 Rust 业务服务；旧前端清理函数只清理 UI 监听器。
4. 修正悬浮窗空闲时的原生可见性，确保字幕模式仍可点击/拖动，听写模式仍不抢焦点。
5. 覆盖启动显示、静默启动、托盘打开、连续点击、窗口已存在、创建失败、多显示器和 DPI 变化。
6. 使用 1.1 采样工具测量前台、托盘 10/60/300 秒、听写后空闲和字幕运行中的工作集、私有字节、CPU 和进程数。
7. 由用户测试关闭动画、托盘恢复、位置尺寸、最大化恢复、快捷键、字幕和 OBS；记录恢复时间与主观延迟。
8. 根据测量决定是否试验指示器空闲销毁；若试验未通过，保留主窗口销毁成果并放弃该附加优化。

## 涉及内容

### 新增文件

- `src-tauri/src/application/window_lifecycle.rs` 或同职责模块：主窗口状态、创建幂等和 ready 队列。
- 生命周期与事件重放测试。

### 修改文件

- `src-tauri/src/desktop/window.rs`：销毁/创建/恢复主窗口，保留可独立回退的 `hide()` 策略。
- `src-tauri/src/desktop/indicator.rs`：原生 hide/show，按测量决定是否增加空闲销毁。
- `src-tauri/src/main.rs`：统一 setup、托盘、关闭事件和窗口 ready 命令。
- `src-tauri/tauri.conf.json`：仅在构建器确需配置调整时修改。
- `ui/src/App.tsx`、`hooks/useTauriBridge.ts`：快照恢复、ready 握手和 UI 监听清理。
- `ui/src/indicator/IndicatorApp.tsx`：仅在需要 ready 握手时调整，不改视觉。

### 可能废弃

- `park_main_window` 的常驻隐藏语义；建议保留为可配置回退函数到 5.1。
- 前端 `beforeunload` 中停止听写/字幕/对比的业务清理。

## 特殊风险与回退要求

- 创建/销毁是桌面窗口高风险改动，必须单独提交；出现阻塞时可只回退该提交，不撤销前两项业务迁移。
- 连续托盘点击必须幂等，不能创建多个 `main` 窗口。
- 窗口未 ready 时不得丢失正在录音、识别或字幕状态。
- 涉及实际窗口点击与拖动的验证只交给用户执行。
