# 3.1 迁移音频会话与听写运行时

## 任务状态

- 状态：已完成
- 所属阶段：第三阶段-后台实时核心迁移
- 前置任务：1.2、2.2
- 关联任务：3.2、3.3、4.2
- 是否可以并行：否；先建立共享音频会话，再供字幕与对比复用

## 任务目标

把麦克风生命周期、全局快捷键处理、实时/文件听写状态机、静音断流、ASR 收尾、本地规则、提示音和文本注入迁到 Rust。完成后即使主 WebView 不存在，全局快捷键仍能完整完成一次听写并正确更新悬浮指示窗。

## 当前情况

- Rust 已负责全局键盘钩子、CPAL 麦克风、ASR 连接器、音频 DSP 和文字注入，但主要以命令和事件形式被前端串联。
- `features/dictation/` 约 1600 行：`session.ts` 保存模块级状态，`realtimeFlow.ts` 管理静音门控与收尾，`fileFlow.ts` 管理录音后文件识别，`micSession.ts` 管理麦克风延迟释放，`hotkeys.ts` 仍有前端焦点回退。
- `localRulesEngine.ts` 使用 JavaScript RegExp；`localRules.worker.ts` 提供超时隔离。
- `lib/cues.ts` 用 Web Audio 播放内置/自定义提示音，自定义数据存于 `localStorage`。
- `indicatorBridge.ts` 负责文本节流和波形摘要，这部分一半是领域节流、一半是展示协议。

## 实施方案

### 共享音频会话

在应用层建立音频会话协调器，明确麦克风的 owner（听写、字幕、对比、音频调试）、附加消费者、延迟释放和冲突规则。继续复用现有 `backend_mic.rs` worker，不在本任务重写 CPAL。

### 听写状态机

用显式 Rust 枚举表达 Idle、WaitingForVoice、Recording、Finishing、ProcessingFile、Injecting、Failed 等状态。每次启动生成 session/epoch；所有 ASR、计时器和音频事件必须校验当前代次。快捷键钩子直接调用应用服务，不再依赖 `app.emit("dictation-toggle")` 被主窗口接收。

### 规则与提示音

- 本地规则必须先确定 JS 正则兼容策略。候选方案需用默认规则、旧内置规则和用户可导出的自定义规则验证；任何不支持的规则都要在保存或迁移时明确提示，不得静默跳过。
- 内置 beep 优先用现有 CPAL 生成；自定义提示音从 1.2 导入的应用数据文件读取，优先复用 Symphonia 解码。若必须新增播放依赖，先记录体积和许可。

### 前端边界

前端保留快捷键设置捕获界面、状态展示、日志展示和指示器 React 渲染。所有开始/停止/取消动作调用 Rust；store 只应用后端状态事件。波形事件由 Rust 先做限频和摘要，禁止继续把完整 PCM base64 广播给主窗口。

## 执行步骤

1. 为当前听写行为补状态机测试用例：切换模式、按住说话、取消、静音断流重连、短音频识别、迟到结果、失败清理和麦克风延迟释放。
2. 设计并实现音频会话协调器，接管 owner、采样率、attach/pause/release 和延迟释放；现有命令先作为适配器调用它。
3. 实现 Rust 听写服务，将 `dictSession`、busy/epoch/timer 和实时/文件分支迁入；使用 2.2 的供应商能力。
4. 将全局快捷键回调改为直接调用听写服务；设置页捕获模式和锁定键特殊处理保持现有行为。
5. 迁移本地规则执行，完成正则兼容决定与回归测试；保留执行超时或复杂度保护。
6. 迁移内置和自定义提示音播放，确认录音设备与输出设备不会死锁。
7. 迁移文本注入前的最终处理与错误传播，确保只注入一次，剪贴板恢复语义不变。
8. 后端直接更新指示器状态、文本和波形摘要；前端听写 controller 切换为命令代理与状态订阅。
9. 验证主窗口隐藏、最小化以及临时销毁测试环境下的完整听写；本任务尚不修改正式关闭策略。

## 特殊风险与回退要求

- 本地规则、提示音、按住说话和锁定键是高风险兼容项；任一无法无损迁移时，将任务标记已阻塞并保留前端路径，不得强行降级。
- 迁移切换点必须保证 Rust 与前端不能同时响应快捷键。
- 所有后台任务退出时必须释放/交还音频 owner；不能依赖 WebView `beforeunload` 清理。
