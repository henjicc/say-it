# 业务后端化与托盘轻量化 · 任务总览

## 任务目标

当前主窗口和悬浮指示窗口各自持有 WebView2 渲染进程，听写、实时字幕、模型对比等运行时又主要由前端 TypeScript 编排，因此关闭主窗口时只能隐藏 WebView，无法真正释放主要内存。本任务在保留 Tauri 2 + React 现有界面和使用体验的前提下，把配置、状态、供应商能力和可后台运行的业务编排逐步迁到 Rust，使主窗口 WebView 可以在进入托盘后安全销毁并按需重建。

本次包含：后端应用状态与事件契约、前端 `localStorage` 数据迁移、模型与供应商能力后端化、听写/字幕/转写/对比/音频调试业务迁移、托盘轻量模式和最终前端职责收敛。不包含：改用 Slint 等新 GUI 框架、重做现有视觉设计、无关功能扩展、直接接入新的供应商或模型。

## 重要说明

- 保留 Tauri + React，不在本任务中更换 GUI 框架；内存目标主要通过主 WebView 按需销毁实现。
- 每个领域迁移期间只能有一个运行时权威，禁止 Rust 与 TypeScript 同时驱动同一会话。
- 前端最终保留视图渲染、表单草稿、画布与高频指针交互；持久化、领域校验、业务状态机、平台能力和后台任务归 Rust。
- 旧 `localStorage` 数据在总体验收前不得删除，必须能回退到迁移前版本。
- 涉及快捷键、鼠标、文字注入、字幕窗口和 OBS 的验证均由用户按任务文件步骤执行。
- 用户已决定所有人工测试统一推迟到全部任务完成后执行；项目期间持续汇总到 `manual-test-checklist.md`，不作为中间阶段门禁。
- 开始 1.1 前应核对 `docs/task/大文件拆分与遗留问题修复/` 和 `docs/task/录音识别与文稿对齐/` 中仍为“待验证”的交互项，避免在未知基线上迁移。
