# 业务后端化与托盘轻量化 · 重要记录

## 记录 001：保留 Tauri 与 React，不进行 GUI 全量重写

- 日期：2026-07-13
- 状态：已确定
- 背景：WebView2 是当前内存主要来源，但全量迁移 Slint 需要重写约 1.28 万行前端、字幕编辑器、Canvas、中文输入和窗口交互，风险远高于单纯内存收益。
- 最终决定：本计划保留 Tauri 2 + React 19，通过业务后端化和主 WebView 按需销毁降低托盘内存。
- 选择原因：可以分领域实施和回退，同时保留已经完成的 UI 系统与复杂编辑器。
- 影响任务：全部任务。

## 记录 002：Rust 是领域状态唯一权威，前端是状态投影

- 日期：2026-07-13
- 状态：已确定
- 背景：当前 TypeScript 模块级状态使 WebView 销毁后无法继续后台业务或恢复会话。
- 最终决定：迁移完成的领域由 Rust 保存配置、运行状态和生命周期；前端通过完整快照和带版本的事件更新 Zustand 只读缓存。
- 选择原因：消除窗口生命周期对业务的控制，并为未来其他 GUI 或无界面运行保留可能性。
- 影响任务：1.1、1.2、3.1、3.2、3.3、4.1、4.2、5.1。

## 记录 003：迁移期禁止同一领域新旧状态机并行

- 日期：2026-07-13
- 状态：已确定
- 背景：快捷键、麦克风和 ASR 会话若被 Rust 与前端同时消费，会造成双启动、双注入和资源冲突。
- 最终决定：每个领域以独立所有权切换完成迁移；配置允许兼容双写，运行时只允许单一权威。
- 选择原因：降低渐进迁移最危险的竞态风险。
- 影响任务：3.1、3.2、4.1、4.2、5.1。

## 记录 004：旧 localStorage 在总体验收前保留

- 日期：2026-07-13
- 状态：已确定
- 背景：听写、字幕、对比、主题和提示音配置尚未进入 `say-it-state.json`；直接删除会破坏用户数据和旧版本回退。
- 最终决定：由前端执行一次性导入，迁移期保留旧键并在界面修改配置时维护兼容镜像；是否在未来版本清理旧键另行决定。
- 选择原因：保证升级和回退都不丢设置。
- 影响任务：1.2、3.1、3.2、4.2、5.1。

## 记录 005：悬浮指示窗先保留 WebView 实现

- 日期：2026-07-13
- 状态：已确定
- 背景：悬浮窗单独占用一个 renderer，但它承担透明、置顶、波形和字幕动画；贸然销毁会引入页面 ready 竞态和首次显示延迟。
- 最终决定：3.3 先销毁主 WebView并测量。只有内存未达目标时，才试验悬浮窗空闲销毁；若影响快捷键或字幕体验则取消该优化。
- 选择原因：主 renderer 是更大且风险更低的优化对象，先获取主要收益。
- 影响任务：3.3、5.1。

## 记录 006：性能验收阈值

- 日期：2026-07-13
- 状态：待确认
- 背景：当前截图中托盘前后缺少统一采样方法，任务管理器“应用/后台进程”分组也不能代表真实内存压力。
- 最终决定：在 1.1 建立进程树采样脚本后确定。当前建议：关闭到托盘 60 秒后的总工作集相对现状下降至少 40%，恢复主窗口和首次快捷键响应不得出现可感知退化。
- 选择原因：先测量基线再锁定目标，避免使用单次任务管理器截图作结论。
- 影响任务：1.1、3.3、5.1。

## 记录 007：本地规则和自定义提示音的 Rust 兼容方案

- 日期：2026-07-13
- 状态：待确认
- 背景：本地规则当前使用 JavaScript RegExp，Rust 常用正则引擎不支持全部 JS 语义；自定义提示音存为 `localStorage` Data URL 并由 Web Audio 播放。
- 最终决定：3.1 开始前，用默认规则、历史升级规则和可收集的自定义规则样本验证候选正则方案；提示音优先复用现有 CPAL/Symphonia 能力，无法无损兼容时再评估新增依赖。不得静默改变规则或声音行为。
- 选择原因：这两项是听写完全脱离 WebView 的兼容关键点，不能在计划阶段臆定具体库。
- 影响任务：1.2、3.1。

## 记录 008：关闭主窗口时各业务的生命周期

- 日期：2026-07-13
- 状态：已确定
- 背景：当前 `hide()` 保留全部前端任务。未来销毁主 WebView 后，听写和字幕必须继续；音频试听等强界面相关行为是否继续需要明确。
- 最终决定：后台听写、字幕、翻译、OBS、文件转写、文稿对齐计算和模型对比继续运行；仅停止依赖可见界面的临时试听、波形预览和未提交草稿。关闭前须提示或安全停止临时试听。
- 选择原因：保持现有后台能力，同时避免不可见的试听持续占用音频设备。
- 影响任务：1.1、3.3、4.1、4.2。

## 记录 009：快照重建与事件版本契约

- 日期：2026-07-13
- 状态：已确定
- 背景：窗口重建期间可能在拉取快照和建立监听之间丢失事件，迟到事件也可能覆盖新会话。
- 最终决定：前端按“拉快照 → 订阅 → 再拉快照校正 revision”重建；领域事件包含递增 revision、domain、eventType、可选 sessionId 和 payload。
- 选择原因：在不引入事件持久队列的前提下提供明确、可测试的重建语义。
- 影响任务：1.2 及所有领域迁移任务。

## 记录 010：配置领域渐进强类型化

- 日期：2026-07-13
- 状态：已确定
- 背景：历史 localStorage 允许自定义规则及未来版本字段；运行时尚未迁移时一次性严格反序列化可能造成未知字段丢失。
- 最终决定：1.2 由 Rust 以版本化、分领域 JSON 对象拥有持久化权威并校验对象边界；具体字段强类型和业务校验随对应领域接管逐步收紧。
- 选择原因：先完成无损所有权迁移，再在单一领域切换时验证语义，避免同时改变配置和运行时行为。
- 影响任务：1.2、2.1、2.2、3.1、3.2、4.2、5.1。
