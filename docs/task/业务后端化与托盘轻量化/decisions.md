# 决策

## 2026-07-13 · 1.1

- Rust 类型为快照与领域事件契约源，TypeScript 保留人工镜像，由 Rust 序列化测试守住关键字段。
- 当前未迁移领域在快照中明确标为 `frontendOwned`，不伪装为后端权威。
- 快照 revision 从后端原子计数器读取；后续状态变更通过统一递增函数推进。
- 生命周期和性能口径以 `01-实施方案.md` 与 `重要记录.md` 为准。
- 两个既有任务的待验证状态不修改：本阶段没有获得用户真实测试结果。
- 用户决定所有人工测试统一推迟到全部任务完成后执行；中间任务以自动验证和静态审查作为阶段门禁，人工项目持续追加到 `manual-test-checklist.md`，最终验收时一次执行。

## 2026-07-13 · 1.2

- `say-it-state.json` schema 版本为 1；未知领域字段按完整 JSON 对象无损保存，由前端既有归一化维持兼容，Rust 负责对象边界、所有权和持久化事务。
- 旧 WebView 数据只在后端未导入时合并；重复启动不覆盖后端新值，显式 retry 也仅补齐空领域。
- 配置先写临时文件并刷新，再备份并替换；提交失败时尝试恢复备份，主文件损坏时读取备份，主备均坏则明确报错。
- 自定义提示音导入应用数据目录 `cues/`，快照只保存相对路径和 MIME 元数据；3.1 前保留旧 Data URL 供 Web Audio 回退播放。

## 2026-07-13 · 2.1

- `shared/asr-models.json` 继续作为仓库数据源，但仅由 Rust 编译期内嵌、解析和校验；前端运行时只消费 `get_model_catalog`。
- 目录独立于应用快照并带版本号，避免静态元数据随运行状态快照重复传输。
- 前端保留同名薄查询和稳定数组引用以控制改动范围，但协议族、通路和能力判断全部来自后端结果。

## 2026-07-13 · 2.2

- 仅有一个真实供应商时采用按 kind 构造的小型枚举 facade，避免为动态插件引入 `async-trait` 或额外依赖；新增供应商只扩展能力工厂和 provider 内部实现。
- 文件识别公共边界保留提交、查询、获取结果等生命周期操作，但 task ID 和上传 URL 只作为 opaque 值传递，不在业务层解释供应商协议。
- customization 是可选能力；不支持时由工厂返回带 provider ID、capability 和 message 的结构化错误，业务层不按供应商名分支。
- 旧 `funasr_*` 命令保留到 5.1，内部只代理通用命令；前端现已调用携带 provider ID 的通用入口。
