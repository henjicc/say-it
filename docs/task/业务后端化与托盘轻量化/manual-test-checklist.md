# 最终人工测试清单

## 执行约定

- 用户决定：所有人工测试统一在全部任务完成后执行，不作为中间任务或阶段门禁。
- 后续任务发现新的鼠标、快捷键、真实音频、窗口、OBS 或性能验证项时，继续追加到本文件。
- 每项执行后记录：通过/失败、测试时间、版本/提交、模型与音源、错误信息或截图；失败须判断是迁移前既有问题还是本次回归。

## 功能回归

- [ ] 实时听写：说一句并停止，最终文本只注入一次；中途取消不注入且指示窗恢复。
- [ ] 文件听写：Fun-ASR-Flash、Qwen3-ASR-Flash 各执行一次且只注入一次；本地规则顺序和替换结果不变。
- [ ] 实时字幕：麦克风与系统音频分别持续滚动；有 OBS 环境时输出同步。
- [ ] 设置页静音测试：成功路径完成且页面状态恢复。
- [ ] 录音识别：真实音频完成转写；纯文本/字幕切换、复制、SRT 导出及播放器时间抽查正常。
- [ ] 文稿对齐：真实音频和逐行文稿执行；两种结果、缓存复用、SRT 文本与时间正常。
- [ ] 字幕编辑器：两个入口均验证 cue 整体/边缘拖动、吸附、缩放、播放头、文本编辑、±100ms 微调，输入框快捷键不误触发。

## 资源与时延

- [ ] 实时字幕运行中执行：`powershell -File scripts/测量进程内存.ps1 -Condition subtitles-running -OutputPath docs/task/业务后端化与托盘轻量化/baseline-subtitles.json`。
- [ ] 托盘点击到主窗口可交互测 5 次，记录每次毫秒数和中位数。

# 3.1 音频会话与听写运行时（实现完成后执行）

- [ ] 完全退出主窗口监听后，分别用切换模式和长按模式完成开始、停止、Esc 取消；最终文本只注入一次。
- [ ] 实时听写开启静音断流，先说话、静音超过阈值、再次说话，确认自动重连且两段文本均保留。
- [ ] 分别使用实时模型和文件模型验证成功、失败、快速取消、极短音频及停止后的迟到结果不污染下一会话。
- [ ] 逐项核对所有本地规则的顺序、开关、替换语义和非法规则提示；灾难性正则不得卡住应用。
- [ ] 内置与自定义开始/结束提示音均可播放，重启后仍有效，录音输入与播放输出无死锁。
- [ ] 主窗口隐藏、最小化和临时销毁三种情况下，指示窗状态、文本和波形摘要正常，主窗口不再收到完整 PCM。
- [ ] 长按模式快速点按（短于 260ms）不启动；超过阈值只启动一次，松开只停止一次；CapsLock/NumLock/ScrollLock 绑定时系统锁定状态不误切换。
- [ ] 听写结束后在麦克风保活时间内再次启动可复用设备；保活到期会释放；期间启动字幕/对比时不会被旧听写计时器误关闭设备。

- [ ] 自定义提示音文件损坏、丢失或输出设备不可用时显示明确错误，不静默改播内置音；设置页试听走系统原生输出。
- [ ] 完成文件听写后检查临时 WAV 不影响下一次会话；极短/空录音明确提示且不注入旧文本。

# 3.2 实时字幕、翻译与 OBS 编排（实现完成后执行）

- [ ] 主窗口正常、隐藏及 3.3 实现的销毁状态下，分别用按钮和全局快捷键开始/停止字幕；悬浮窗持续更新，重建主窗口后运行状态和最新原文正确恢复。
- [ ] 麦克风与系统音频各持续识别至少 3 分钟；拔插/切换指定设备后行为明确，音源冲突时不得同时抢占听写或模型对比的设备。
- [ ] 开启字幕静音断流：有声连接、静音超过阈值后等待、再次有声重连；连续网络中断超过最大重试后显示明确错误且音频资源可被下一会话重新取得。
- [ ] 替换模式验证 2.5 秒内停顿续接、超过 2.5 秒清空重开及 1800 字符裁剪；滚动模式分别设置 1～4 行，确认只显示最近行且动画无跳行。
- [ ] 开启翻译后，用含句号、两个逗号、无标点长句的语音核对分句延迟；分别测试增量/非增量模型，停止重开后旧译文不得进入新会话。
- [ ] OBS 安装并连接 Browser Source 后开启输出：桌面悬浮窗隐藏、原文/译文/布局实时同步；短暂断线不闪回，持续断线约 2 秒后回落桌面，重连后恢复 OBS。
- [ ] 字幕运行中修改字体、字号、行数、双语/仅译文、原译顺序及 OBS 路由，悬浮窗和 OBS 布局同步且不重启音频/ASR 会话。
- [ ] 字幕预览开关、离开页面和样式调整均正常；预览不启动麦克风、不产生翻译请求、不写 OBS，关闭预览不停止已在后台运行的真实字幕。
- [ ] 在字幕悬浮窗上拖动并关闭：拖动位置正常，关闭请求只停止真实字幕一次；窗口重建后不出现预览计时器或旧译文串场。

- [ ] 调整主窗口位置和内容区尺寸后关闭/恢复；再分别验证最大化关闭/恢复、最小化后关闭/恢复，尺寸不应逐次变大，最大化状态应保持。
