# 3.3 端到端验证与经验沉淀

## 任务状态

- 状态：待开始
- 所属阶段：第三阶段-文稿对齐
- 前置任务：2.3、3.2
- 关联任务：无
- 是否可以并行：否（收尾任务）

## 任务目标

两条功能线（录音转写、文稿对齐）在接近真实使用的场景下完整验证；既有功能回归；有价值的坑写入 `docs/experience`；收尾计划文档。

## 当前情况

- 各任务已做过局部验证，本任务做交叉与边界场景的系统验证，以及构建级检查。

## 实施方案 / 执行步骤

1. **构建检查**：`npx tsc --noEmit`、`cargo check`、`npm run ui:build` 均通过。
2. **录音转写场景**（用户配合执行）：
   - 长音频（≥30 分钟）完整流程与轮询耐心度；
   - 视频容器（mp4/mkv）直接识别；
   - 开启说话人分离的双人对话音频；
   - 网络异常（识别中断网）与恢复后重试。
3. **文稿对齐场景**（用户配合执行）：
   - 口播视频 + 原稿（典型用例）导出 SRT 实际挂载核对；
   - 含数字、英文品牌名的文稿匹配率表现；
   - 文稿行数远多于/少于实际语音时的行为。
4. **回归**：语音输入、实时字幕、设置页热词与高级参数各冒烟一次。
5. **经验沉淀**：将过程中的非显而易见问题（如 oss:// 域名兼容、参数陷阱、对齐边界 case）按仓库约定各自写入 `docs/experience/` 独立文件。
6. **收尾**：更新 `00-任务总览.md`（全部状态、最近记录）；`重要记录.md` 中「待确认」项全部落定或标注遗留。

## 涉及内容

- 无新增业务代码；仅可能修复验证中发现的缺陷（修复超出小改动时新增任务，不在本任务内扩散）。
- `docs/experience/*.md`：经验文件。
