## 任务目标

Rust 实现纯算法模块与命令 `align_transcript`：输入 ASR 词级时间戳序列 + 用户文稿（一行一句），输出行级字幕（begin/end 毫秒、文本 = 原文稿行、每行匹配率），时间轴单调不重叠。

## 当前情况

- 输入数据来自 1.2 的 `TranscriptionResult`（所有 `sentences[].words[]` 拍平即得词序列，每词含 `begin_time`/`end_time`/`text`/`punctuation`）。
- 核心难点：文稿与识别结果存在插入/删除/替换差异（口误、语气词、数字读法、识别错误），需要鲁棒对齐。算法思路已定稿，见 [01-实施方案.md](../../01-实施方案.md) 第 4.4 节，本文件为实现细化。
- 项目无现成文本算法代码；纯计算逻辑放独立模块，不依赖 tauri。
- 参考关键词（需要更多资料时搜索）：forced alignment、text-audio alignment、Needleman-Wunsch、anchor-based sequence alignment；同类实现：aeneas、stable-ts 的 align 模式。

## 实施方案

新建 `src-tauri/src/text_align.rs`：

1. **Token 化**：`tokenize(text) -> Vec<Token>`。NFKC 归一（可用简化实现：全角→半角、大写→小写）、跳过标点/空白；CJK 每字一个 token，连续拉丁/数字为一个 token。文稿侧 Token 记录 `line_index`；ASR 侧 Token 记录时间（多字词内部按字数在词时间段内线性内插）。
2. **对齐**：`align(script_tokens, asr_tokens) -> Vec<Option<usize>>`（文稿 token → ASR token 下标）。
   - 锚点：对两侧做 n-gram（n=5，退化到 3）哈希，取两侧均唯一且匹配的 n-gram 为锚点，按位置单调筛选（最长递增子序列保证锚点不交叉）。
   - 锚点之间的段用 Needleman-Wunsch（match=+2、mismatch=-1、gap=-1）；段长超过阈值（如 2000×2000）时用带宽限制 DP 兜底防止内存爆炸。
3. **行时间**：每行 begin/end 取该行首/尾匹配 token 的时间；无匹配行按相邻匹配边界内插；后处理保证单调、不重叠（相邻冲突时在中点截断）、最小时长 300ms。
4. **输出**：`AlignedLine { line_index, text, begin_ms, end_ms, match_ratio, interpolated: bool }`。
5. 命令 `align_transcript(words, script) -> Vec<AlignedLine>` 放入 `commands/transcription.rs`（同步命令即可，几万 token 毫秒级完成；若实测慢再改 async）。
6. **单元测试**（本任务必须有，`#[cfg(test)]`）：完全一致、文稿多字/少字、识别错字、整行不匹配（内插）、中英混排、空输入。

### 设计定稿（实现时在 4.4 基础上的细化与强化）

1. **数字字符等价**：规整时把中文数字字符（〇零一二三四五六七八九、两）映射为阿拉伯数字，阿拉伯数字串逐字符切 token。解决“2024”vs“二零二四”类读法差异（“两千零二十四”也能部分命中：2/0/2/4 匹配、千/十走 gap）。只做字符级等价，不做完整数字归一化。
2. **半全局对齐**：整体对齐两端 ASR 侧 gap 免罚（免罚只作用于最外层段的首/尾），容忍音频中存在文稿之外的片头/片尾内容（闲聊、口播等），文稿可只对齐到音频中的一个窗口。
3. **替换对参与计时**：NW 对角替换（识别错字）位置大概率正确，参与行边界计时；`match_ratio` 只统计真匹配。
4. **行命中可靠性判定**：行内命中（匹配+替换）≥2 且 ≥20% 行 token 数，或 ≤3 token 的短行有真匹配，才信任自身命中取时间；否则按相邻行边界 + token 数加权内插并标 `interpolated`，防止 CJK 常见字（的/了/是）在差异区随机配对导致边界漂移。
5. **规模控制**：段积 ≤ 400 万格直接全矩阵对齐；更大段用唯一 n-gram 锚点（5→3）+ 最长递增子序列筛选 + 显式栈迭代分治（防递归爆栈）；完全无锚点时带宽 DP 兜底（带宽 = 两侧长度差 + 128，总格数硬上限 6400 万）。
6. **空行/纯标点行**：0 token 行按内插得到零宽时间点、`match_ratio=0`、不施加最小时长；3.2 界面导出 SRT 前应过滤空行。
7. **仿射 gap 计分（Gotoh 三状态，替代原 match=+2/mismatch=-1/gap=-1 线性计分）**：match=+8、mismatch=-4、gap 开口=-6、延续=-1。原因：线性 gap 下说话人大段即兴（长插入）时，对齐宁可把文稿尾部“就近替换”到无关内容上（配合尾部免罚零成本），也不付长 gap 的代价命中其后的真实匹配，导致行尾时间错误；仿射后长 gap 便宜，跳过插入命中真实匹配；且整段无关区域倾向干净的双侧 gap 而非乱配替换对，内插判定更准。
8. **双向相似度 `similarity`（Dice）**：`2×匹配 / (行 token + 命中区间内 ASR token)`。与 `match_ratio` 的区别：行全部匹配但区间内音频多说了大量内容时也会显著下降。同时输出行命中的 ASR 词范围 `asr_word_begin/end`（输入 words 下标），供“识别修正”版结果按保留行边界切出替换区间（见 3.2）。
