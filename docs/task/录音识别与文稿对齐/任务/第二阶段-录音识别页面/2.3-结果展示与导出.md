## 任务目标

识别完成后提供两种结果视图：**纯文本**（可全选复制）与**带时间戳字幕**（逐条 cue：序号、时间段、文本，说话人分离时显示说话人标记）。支持复制全文与导出标准 SRT 文件。

## 当前情况

- 结果数据结构（`transcripts[].sentences[].words[]`，毫秒时间戳）由 1.2 定义并已进入 store（2.2）。
- 结果 JSON 自带句级切分 `sentences[]`，但单句可能过长不适合做字幕，需要用词级时间戳二次切分——策略已定，见 [01-实施方案.md](../../01-实施方案.md) 第 4.3 节。
- 文件保存对话框：2.1 引入的 `@tauri-apps/plugin-dialog` 的 `save()`；写文件项目暂无能力，需新增一个简单后端命令（不引入 `tauri-plugin-fs`，见 重要记录 004）。

## 实施方案

- 纯函数模块 `ui/src/features/transcription/subtitles.ts`：
  - `buildCues(result, options) -> Cue[]`：以 `sentences` 为基础；超长句（默认 CJK 折算宽度 > 30，英文单词按 2 字宽估算）在词级切分——优先词后带标点处，其次相邻词时间间隙最大处；相邻短句可合并（间隙 < 500ms 且合并后不超宽）。`Cue = { index, beginMs, endMs, text, speakerId? }`。
  - `toSrt(cues) -> string`：`HH:MM:SS,mmm` 格式，CRLF，空行分隔。
  - 该模块 3.2 会复用 `toSrt` 与 cue 渲染，保持无状态。
- 视图：结果区顶部小 Tabs/Pill 切换「纯文本 / 字幕」；纯文本用 `transcripts[].text` 拼接；字幕视图逐条渲染（等宽时间 + 文本），说话人分离时前缀「说话人 N」。
- 操作：「复制文本」「导出 SRT」按钮；导出走 dialog `save()`（默认文件名 = 音频文件名 + `.srt`）+ 新命令 `save_text_file(path, content)`（后端校验路径非空、写 UTF-8，错误传播）。
- 多音轨（`transcripts` 多元素）场景：当前仅识别第一音轨（channel_id 默认 [0]），按单 transcript 处理，代码上取第一个并容错。

## 执行步骤

1. 实现 `subtitles.ts`（`buildCues` / `toSrt`），处理空结果、无 words、时间缺失等边界。
2. 后端新增 `save_text_file` 命令（`commands/common.rs` 或 `commands/transcription.rs`），注册到 `main.rs` 与 `CMD`。
3. 视图实现双模式展示与复制/导出；导出成功给出提示（含保存路径）。
4. `npx tsc --noEmit`、`cargo check`；真实音频识别后导出 SRT，用播放器（如 PotPlayer 挂载）或文本检查时间格式。
5. 交给用户实测：两种视图切换、复制、导出后在播放器中查看字幕时间是否对齐。
