# 2.2 识别流程与参数面板

## 任务状态

- 状态：待开始
- 所属阶段：第二阶段-录音识别页面
- 前置任务：1.2、2.1
- 关联任务：2.3（展示本任务产出的结果）、3.2（复用执行流程）
- 是否可以并行：否

## 任务目标

录音转写 Tab 可实际执行：选中文件 → 点击「开始识别」→ 展示阶段进度（上传中 / 识别中 / 完成 / 失败）→ 结果数据进入 store。参数面板默认折叠、可修改、持久化。

## 当前情况

- 后端命令 `transcription_start` / `transcription_cancel` 与事件 `transcription-event` 由 1.2 提供（事件 payload 结构见 1.2 任务文件）。
- 事件监听范式：`ui/src/lib/tauri.ts` 的 `on()`；流程编排范式参考 `ui/src/features/dictation/controller.ts` / `features/subtitles/controller.ts`（模块级 controller + zustand store）。
- 参数持久化：`funasr` profile 的 `config` 经 `useProviderStore.updateConfig`（后端 `update_provider_config` 为浅合并 patch，见 `commands/funasr.rs::apply_funasr_patch`——注意它是 funasr 专用 patch 函数，本任务直接用通用的 `update_provider_config` 命令即可）。参数面板样式范例：`views/SettingsProviderPanel.tsx` 的嵌套 `Collapse`。
- 未配置 API Key 时后端会报错；页面需在开始前检查 `useProviderStore` 的 `hasApiKey` 并引导去设置页。

## 实施方案

- `features/transcription/controller.ts`：`startTranscription()` 读取文件与参数 → invoke `transcription_start` → 订阅 `transcription-event` 按 `jobId` 过滤 → 更新 store 的 `stage`/`result`/`error`；`cancelTranscription()`。
- store 增加：`params`（见下）、`stage: "idle" | "uploading" | "recognizing" | "completed" | "error"`、`result`、`errorMessage`。
- 参数面板（`Collapse` 默认收起，标题「识别参数」）：
  - 模型版本：下拉，选项 `fun-asr`（默认，推荐）、`fun-asr-2025-11-07`、`fun-asr-2025-08-25`、`fun-asr-mtl`、`fun-asr-mtl-2025-08-25`。
  - 语种提示 `language_hints`：与设置页一致的多选（中文/英文/日语，另加“自动”即为空）。
  - 说话人分离 `diarization_enabled` 开关 + `speaker_count`（2~100，仅开启时可填，留空自动）。
  - 热词 ID `vocabulary_id`：文本框，默认空；旁注说明与实时识别的热词列表不通用（见 重要记录 003）。
  - 参数保存进 `funasr` config 的 `transcription` 子对象（如 `{ transcription: { model, languageHints, diarizationEnabled, speakerCount, vocabularyId } }`），页面加载时回读。
- 进度 UI：阶段文案 + 转圈；识别中提供「取消」；失败显示中文错误与「重试」。

## 执行步骤

1. store 扩展参数与阶段字段；页面加载时从 `useProviderStore` 的 `funasr.config.transcription` 回读参数。
2. 实现参数面板 UI（折叠、控件、保存：参数变更即调用 `updateConfig` 防抖保存，风格与设置页一致）。
3. 实现 controller 的开始/取消/事件订阅，接通按钮与进度展示；未配置 API Key 时给出提示与跳转设置的引导。
4. `npx tsc --noEmit` 通过；`npm run tauri:dev` 下用真实短音频跑一次完整流程。
5. 交给用户实测：正常识别、识别中取消、无 API Key 提示、参数修改后重启应用仍保留。
