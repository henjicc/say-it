# 2.1 页面框架与文件选择

## 任务状态

- 状态：待开始
- 所属阶段：第二阶段-录音识别页面
- 前置任务：无
- 关联任务：2.2、3.2（在本框架上继续开发）
- 是否可以并行：可与第一阶段并行

## 任务目标

左侧导航出现「录音识别」入口；页面含「录音转写 / 文稿对齐」两个 Tab；录音转写 Tab 可通过文件对话框或拖放选中一个本地音视频文件并显示文件信息（名称、大小、格式校验提示）。本任务不接识别流程。

## 当前情况

- 导航：`ui/src/components/shell/navConfig.tsx`（`NAV_ITEMS` + 图标 svg）；`ViewKey` 在 `ui/src/store/useUiStore.ts`（当前 `"dictation" | "subtitles" | "settings"`）；视图映射在 `ui/src/App.tsx` 的 `VIEWS`。
- 可复用组件：`Tabs`（`ui/src/components/ui/Tabs.tsx`）、`Card`、`Button`、`Collapse` 等；页面布局参考 `views/RealtimeSubtitlesPanel.tsx` / `DictationView.tsx` 的写法。
- 项目**没有**文件对话框能力，需新增 `tauri-plugin-dialog`（Rust crate + npm 包 `@tauri-apps/plugin-dialog`，官方插件，见 重要记录 004）；权限需在 `src-tauri/capabilities/` 下现有 capability 文件中授权（先查看该目录现状再改）。
- 拖放：Tauri 2 webview 的 drag-drop 事件（`@tauri-apps/api` `getCurrentWebview().onDragDropEvent`）可拿到真实文件路径；注意 `tauri.conf.json` 中 `dragDropEnabled` 若被禁用需确认（当前配置未显式禁用，执行时核实）。

## 实施方案

- 新增 `ViewKey` 值 `"transcription"`，导航项「录音识别」（图标：文件+波形风格 svg，风格对齐现有 1.8 stroke 线性图标），插在「实时字幕」之后。
- `views/TranscriptionView.tsx`：顶层 Tabs（录音转写 / 文稿对齐），文稿对齐 Tab 先放占位文案。
- `store/useTranscriptionStore.ts`：本任务先落 `selectedFile: { path, name, size } | null` 与 Tab 状态。
- 文件选择区：大块可点击拖放区域（虚线框），点击走 dialog `open()`（filter 到支持的扩展名），拖放走 onDragDropEvent；选中后显示文件名/大小与「重新选择」；按 3.3 节规格做扩展名与 2GB 大小前端校验（仅提示，不阻断非常见扩展名，服务端为准）。

## 执行步骤

1. `src-tauri`：`Cargo.toml` 加 `tauri-plugin-dialog = "2"`，`main.rs` 注册 `.plugin(tauri_plugin_dialog::init())`；检查并更新 `src-tauri/capabilities/` 授权。
2. 前端：`npm install @tauri-apps/plugin-dialog`。
3. `useUiStore.ts` 增加 `"transcription"`；`navConfig.tsx` 增加导航项；`App.tsx` 注册视图。
4. 新建 `useTranscriptionStore.ts` 与 `TranscriptionView.tsx`（Tabs + 文件选择区 + 文稿对齐占位）。
5. `npx tsc --noEmit`、`cargo check` 通过，`npm run tauri:dev` 启动确认导航与页面渲染。
6. 交给用户实测：点击选择文件、拖放文件、选择不支持扩展名的文件（应有提示）、切换 Tab。
