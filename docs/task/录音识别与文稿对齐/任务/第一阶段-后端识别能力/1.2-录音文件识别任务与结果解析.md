# 1.2 录音文件识别任务与结果解析

## 任务状态

- 状态：待开始
- 所属阶段：第一阶段-后端识别能力
- 前置任务：1.1
- 关联任务：2.2（前端调用）、3.1（消费词级时间戳）
- 是否可以并行：否（依赖 1.1 的 oss:// URL）

## 任务目标

Rust 侧打通完整识别链路并暴露 Tauri 命令：`transcription_start`（文件路径 + 参数 → 后台异步执行「上传 → 提交任务 → 轮询 → 下载解析结果」，各阶段 emit 事件）与 `transcription_cancel`。产出结构化结果（全文 + 句级 + 词级时间戳）。

## 当前情况

- 接口细节（提交/轮询/结果 JSON 结构、请求头、模型列表）已核实，见 [01-实施方案.md](../../01-实施方案.md) 第 3.2/3.3 节。
- 异步任务 + 事件推送的既有范式：`commands/asr.rs::start_asr_stream`（`tauri::async_runtime::spawn` + `emit_asr_stream_event`）；事件发送函数在 `commands/audio.rs`。
- 命令注册位置：`src-tauri/src/main.rs` 的 `invoke_handler`；模块声明在 `commands/mod.rs`。
- API Key 读取：复用 `commands/funasr.rs::funasr_credentials` 的模式（读 `funasr` profile 的 `apiKey`）。

## 实施方案

- `providers/alibabacloud/transcription.rs`（纯协议）：
  - `TranscriptionParams`（camelCase 反序列化）：`model`（默认 `fun-asr`）、`vocabulary_id`、`language_hints`、`diarization_enabled`、`speaker_count`、`channel_id`；序列化为请求体 `parameters`（空值不下发；`parameters` 至少传 `{}`）。
  - `submit_task(api_key, file_url, params) -> task_id`：POST transcription 端点，头含 `X-DashScope-Async: enable`；`file_url` 以 `oss://` 开头时加 `X-DashScope-OssResourceResolve: enable`。
  - `query_task(api_key, task_id) -> TaskStatus`：解析 `task_status` 与 `results[0]`（含 `subtask_status`/`transcription_url`/`code`/`message`）。
  - `fetch_result(transcription_url) -> TranscriptionResult`：下载 JSON，解析为 `TranscriptionResult { duration_ms, transcripts: [ { channel_id, text, sentences: [ { begin_time, end_time, text, speaker_id?, words: [ { begin_time, end_time, text, punctuation } ] } ] } ] }`（Serialize + camelCase，直接回传前端）。
- `commands/transcription.rs`：
  - `RuntimeState` 增加 `transcriptions: Mutex<HashMap<String, CancelFlag>>`（`Arc<AtomicBool>` 即可）。
  - `transcription_start(app, state, file_path, params) -> { job_id }`：spawn 后台任务，阶段事件 `transcription-event`，payload `{ jobId, stage, ... }`；stage 依次 `uploading`、`submitted`（含 taskId）、`polling`（可含轮询次数）、`completed`（含完整 `TranscriptionResult`）、`error`（含中文 message）。
  - 轮询：首次 2s，之后 4s，总超时 30 分钟；每轮检查取消标志。
  - `transcription_cancel(job_id)`：置取消标志（远端任务不强制取消，说明记录在事件里）。
- 整体 `SUCCEEDED` 但 `subtask_status=FAILED` 时按失败处理，透传 `code`/`message`。

## 执行步骤

1. 新建 `providers/alibabacloud/transcription.rs`，实现参数、提交、查询、结果解析与对应类型；`mod.rs` 导出。
2. 新建 `commands/transcription.rs`，实现 `transcription_start` / `transcription_cancel` 与事件发送；`commands/mod.rs`、`main.rs` 注册；`state.rs` 增加任务表。
3. `ui/src/lib/tauri.ts` 的 `CMD` 增加 `transcriptionStart` / `transcriptionCancel`，`EVT` 增加 `transcriptionEvent`（仅登记名字，页面接入属 2.2）。
4. `cargo check` + `npx tsc --noEmit`。
5. 真机联调（可用临时脚本或 devtools 手动 invoke）：用一个 1 分钟内的真实音频跑通全链路，确认 `completed` 事件里 sentences/words 时间戳合理；同时验证 1.1 的上传（若其推迟验证）。
6. 重点验证风险项：`oss://` URL + 通用域名是否被异步转写接口接受；若报错 `FILE_DOWNLOAD_FAILED` 等，按实施方案第六节回退方案调整并更新文档。
