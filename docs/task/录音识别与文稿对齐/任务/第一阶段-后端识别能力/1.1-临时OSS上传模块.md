# 1.1 临时OSS上传模块

## 任务目标

Rust 侧提供一个函数：输入本地文件路径 + API Key + 模型名，完成「获取上传凭证 → multipart 直传 OSS」，返回 `oss://` 临时 URL。

## 当前情况

- 项目无任何上传能力。HTTP 调用范式参考 [customization.rs](../../../../../src-tauri/src/providers/alibabacloud/customization.rs) 的 `post_customization`（reqwest + Bearer + 错误提取）。
- `src-tauri/Cargo.toml` 中 reqwest features 为 `["default-tls","rustls-tls","gzip","json","stream","http2"]`，**缺少 `multipart`**，需补上。
- API Key 从 `funasr` profile 读取，读取方式见 `commands/funasr.rs::funasr_credentials`。
- 接口事实（URL、字段、约束）已核实并记录在 [01-实施方案.md](../../01-实施方案.md) 第 3.1 节，执行时直接照抄，不需要再查文档。

## 实施方案

新建 `src-tauri/src/providers/alibabacloud/uploads.rs`，纯协议模块（不依赖 tauri State）：

- `get_upload_policy(api_key, model) -> UploadPolicy`：GET `https://dashscope.aliyuncs.com/api/v1/uploads?action=getPolicy&model={model}`，解析 `data` 内字段。
- `upload_file(policy, file_path) -> String`：`reqwest::multipart::Form`，字段顺序 `OSSAccessKeyId`、`Signature`、`policy`、`x-oss-object-acl`、`x-oss-forbid-overwrite`、`key`、`success_action_status=200`、`file`（最后，用 `tokio::fs::File` + stream 避免整读进内存）；返回 `oss://{key}`。
- 组合函数 `upload_for_model(api_key, model, file_path) -> Result<String, String>`；凭证 300 秒有效，取到后立即上传。
- 文件名处理：`key` 中的文件名取原文件名；含特殊字符时做保守清洗（保留扩展名，扩展名用于服务端识别格式）。

## 执行步骤

1. `src-tauri/Cargo.toml` 的 reqwest features 增加 `"multipart"`。
2. 新建 `uploads.rs`，实现 `UploadPolicy` 反序列化结构（字段：`policy`、`signature`、`upload_dir`、`upload_host`、`oss_access_key_id`、`x_oss_object_acl`、`x_oss_forbid_overwrite`、`max_file_size_mb`）与上述三个函数；错误信息用中文并带上游响应摘要（参考 customization.rs 的 truncate 做法）。
3. 在 `providers/alibabacloud/mod.rs` 中 `mod uploads;` 并 `pub use`。
4. 上传前校验文件存在、大小 ≤ `max_file_size_mb`（以接口返回为准），超限返回明确中文错误。
5. `cargo check` 通过。
6. 真机验证：临时加一个调试命令或在 1.2 联调时一并验证（允许推迟到 1.2，但需在执行记录注明）。
