# 录音识别与文稿对齐 · 任务总览

## 任务目标

在「说吧！」左侧导航新增「录音识别」页面，接入阿里云百炼 Fun-ASR **录音文件识别**（异步 HTTP API），复用现有 Fun-ASR 实时识别的同一个 API Key。音频文件通过阿里云百炼**临时 OSS**（getPolicy + 表单直传）上传获得 `oss://` 临时 URL。识别结果支持两种展示：纯文本、带时间戳字幕（可导出 SRT）。在此基础上提供**文稿对齐**功能：用户提供音频 + 一行一句的文稿，用识别结果的词级时间戳与文稿做对齐，生成文本 100% 遵循用户文稿、时间轴精确的字幕。

**属于本次任务**：临时 OSS 上传、录音文件识别任务提交/轮询/结果解析（Rust 后端）、录音识别页面（文件选择、可折叠参数、结果展示、SRT 导出）、文稿对齐算法与界面。

**不属于本次任务**：实时识别/语音输入/实时字幕既有功能的改动、正式 OSS 存储、批量文件队列、字幕编辑器、说话人分离结果的可视化优化（仅透传参数）。

## 重要说明

- 复用 `funasr` 供应商配置里的同一个 `apiKey`（阿里云百炼），不新增密钥。
- 临时 OSS 的 `oss://` URL 有效期 48 小时、上传凭证接口限流 100 QPS，官方注明**不建议生产环境**；本应用为个人桌面工具、单文件低频调用，接受该限制（见 重要记录 002）。
- 现有热词词表以 `target_model=fun-asr-realtime` 创建，对录音文件识别模型 `fun-asr` **可能不生效**，暂不自动复用（见 重要记录 003，待确认）。
- 文件选择需要新增 `tauri-plugin-dialog` 依赖（见 重要记录 004）。
- 执行任何任务前，先阅读本文件、[01-实施方案.md](01-实施方案.md) 和对应任务文件。
