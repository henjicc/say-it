# 模型对比运行时查模型必须同时覆盖内置与插件模型

## 场景

在“设置 > 对比”里选择 JavaScript 插件提供的模型（如豆包 Web、千问 Web）后，单元格直接报“模型未登记”。

## 根因

前端下拉来自 `application/catalog.rs` 生成的动态模型目录，它会合并：

- 内置模型注册表 `providers/registry.rs`
- 已加载插件的 `manifest.models`

但模型对比运行时 `application/compare.rs` 之前只调用 `registry::model_info()` 查询静态内置表，没有回退到 `plugin_registry`，于是：

- UI 能展示并选中插件模型
- 对比页启动时却把插件模型判成“模型未登记”

## 正确做法

凡是“用户可选模型”进入运行时校验、分类或分流逻辑时，都要使用“内置 + 插件”统一解析，而不是只查静态注册表。

对模型对比场景，至少要覆盖：

- 启动前的模型存在性校验
- 实时 / 文件模型分类判断
- 录音停止后文件任务分流
- 上传模式下文件模型筛选

## 经验

只要某个页面的模型列表来自动态目录，就必须检查对应后端运行时是否也读取了同一套模型来源；否则非常容易出现“前端可选、后端不认”的错位。
