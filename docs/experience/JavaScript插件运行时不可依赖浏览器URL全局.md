# JavaScript插件运行时不可依赖浏览器URL全局

## 触发条件

- 说吧的 JavaScript 供应商插件在 QuickJS 宿主里运行。
- 连接器代码直接使用 `URL`、`location`、`window` 等浏览器全局来解析地址或查询参数。

## 正确做法

- 连接器必须只依赖插件 API 文档里明确提供的宿主能力。
- 解析 `wss://`、`https://` 地址时，优先使用纯字符串解析或自行实现最小必需解析逻辑，不要假设 `URL` 全局一定存在。
- 若同一份逻辑既会在网页登录页脚本中运行，又会在连接器里运行，要把“浏览器环境专用逻辑”和“QuickJS 运行时逻辑”拆开。
- 回归测试除了在 Node 下跑通外，还要覆盖“没有 `URL` 全局对象”这类宿主差异。

## 原因

- 网页注入脚本运行在真实浏览器环境里，`URL` 可用；连接器运行在 QuickJS 宿主里，不保证提供浏览器 Web API。
- 这类差异不会在普通 Node 冒烟测试里自然暴露，但会在真实插件运行时表现为“会话明明有效，却被连接器判定为解析失败”。
