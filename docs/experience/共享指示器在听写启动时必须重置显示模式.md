# 共享指示器在听写启动时必须重置显示模式

## 触发条件

语音输入和实时字幕复用 `dictation-indicator` 悬浮窗口，字幕会写入字体、宽度和样式配置。

## 正确做法

听写启动前必须由后端向该窗口发送 `mode: "dictation"`，并清空原文和译文；只恢复默认窗口尺寸不足以重置 React 内存中的字幕样式。这样主窗口销毁或字幕停止后再次听写时，仍会使用听写自身的紧凑样式。

非实时文件听写才发送波形事件；实时听写的识别文本与波形面板不能同时显示。每次切入实时听写时也要显式关闭上一次文件听写留下的波形状态。

非实时文件听写的波形应来自与设置中“录音调整”相同参数的流式 DSP 输出；只把原始 PCM 用于文件识别上传。不能用原始输入绘制波形，否则低音量录音会显示为几乎不波动的点阵，且与调整后的试听不一致。

文件识别完成后的最终文本只应通过领域状态投影并注入目标应用，不能再发送给悬浮指示器；否则隐藏事件与文本事件的异步时序会使最终结果短暂闪现。

提示音属于交互反馈而非听写状态机；内置短提示音应由主 WebView 的持久 Web Audio 上下文播放。不要为每次提示临时创建并销毁原生 CPAL 输出流，否则部分扬声器驱动会在开始或停止处产生破音。

主窗口可关闭时，听写开始与结束提示音必须改由仍常驻的 `dictation-indicator` WebView 播放；首次创建该窗口后要等待其事件监听器就绪再派发提示音事件。
