# 实时字幕悬浮窗：阴影被窗口边界裁切出现硬边

## 现象

字幕框圆角外侧出现一条硬边，尤其在右侧或底部明显，像阴影突然被切断。

## 根因

字幕 WebView 是透明无边框窗口，`body` / `#indicator-root` 使用 `overflow: hidden`。
原实现里字幕窗口宽度基本等于字幕内容宽度，只在 CSS 中给字幕模式留了 `24px` 左右边距和
`24px` 底边距；但字幕框阴影是 `0 18px 48px rgba(...)`，阴影扩散范围大于透明窗口留白，
所以阴影画到原生窗口边界后会被 WebView 裁掉，形成硬边。

## 修复思路

不要只调小阴影掩盖问题，而是把透明窗口当作字幕视觉层的画布：

- 字幕内容宽度仍按用户设置计算；
- 原生窗口宽度额外增加左右阴影安全边距；
- 字幕模式 CSS padding 同步扩大；
- 调整 top / bottom 锚点的窗口偏移，尽量保持字幕框本体原来的屏幕位置；
- 后端窗口尺寸上限要覆盖额外透明留白，否则前端请求的安全边距仍可能被 clamp 掉。

## 经验

透明悬浮窗里的阴影、模糊、发光等视觉效果都必须计入原生窗口尺寸。只看 DOM 元素宽高是不够的；
WebView 边界之外的像素一定会被裁切。
