# 插件适配技能工作区隔离

## 触发条件

使用供应商适配 Skill 生成插件时，当前工作目录就是用户指定的插件工作根目录，且用户只要求生成插件包。

## 正确做法

- 以当前工作目录为默认边界；除非用户明确提供外部路径，Skill 不得访问工作目录外的文件。
- 在当前目录下固定创建 `sayit-plugin-work/<plugin-id>/source`、`build`、`keys` 和 `dist`，不能在根目录随意创建 `plugins/` 等目录。
- 最终产物只能写到 `dist/<plugin-id>-<version>.sayit`，生成完成后交给用户手动通过插件管理安装；除非用户明确要求，不得自动安装。
- 初始化、打包、签名和归档脚本都应接收并校验当前工作根目录，避免 Skill 指令被遗漏时访问或写入外部路径。
