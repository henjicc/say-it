# Windows 原生上下文超时应保留基础窗口元信息

原生文本提取依赖 UIA、IA2 等跨进程调用。Chromium/Electron 的焦点元素查询可能长时间不返回，即使正文探针隔离在子进程中，调用方仍会先到达硬截止。

处理方式：在提交异步正文捕获前，用当前进程同步读取目标 PID、进程名和窗口标题；黑名单仍在这一阶段直接阻止。正文任务超时、探针异常或任务槽忙时，保留这些元信息作为 `Captured` 的降级上下文，而不是返回完全空的 `TimedOut`。已确认密码控件和黑名单不能使用该降级路径。

同时，调用方不得在原生探针之前重复进行全局 `IUIAutomation::GetFocusedElement` 密码检查；密码检查交由隔离探针在读取正文前执行。探针内应预留 IPC 回传余量，并在原生读取预算用尽时跳过后续读取器和深度剪贴板操作。
