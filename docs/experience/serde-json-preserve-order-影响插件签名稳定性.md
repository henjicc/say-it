# serde_json 的 preserve_order 会影响插件签名稳定性

## 触发条件

新增依赖间接启用 `serde_json/preserve_order` 时，Cargo 的特性统一会让整个依赖图中的 `serde_json::Map` 从按键排序改为保留插入顺序。

## 风险

如果签名载荷直接序列化包含 `HashMap` 的结构，同一份语义数据可能因插入顺序不同产生不同字节，导致签名不稳定或验签失败。不能依赖 `serde_json` 默认的对象键顺序作为签名规范。

## 正确做法

生成签名载荷前递归规范化 JSON：对象键按字典序排序，数组顺序保持不变，再执行序列化。新增或升级会影响 `serde_json` 特性的依赖后，必须运行插件签名稳定性测试。
