# 持久化值正确但自定义下拉框回显错误

## 排查顺序

设置重启后显示异常时，先读取 WebView Local Storage 的实际值，区分“未保存”和“控件回显错误”。不要只根据界面显示修改持久化逻辑。

## 本次结论

最初只检查了 `http://localhost:5155` 开发源的 Local Storage，并据此误判为自定义 `Select` 回显错误。Tauri 开发页、打包页和不同 WebView 可能使用不同 origin，单个 origin 的 Local Storage 不能作为应用级设置已正确恢复的证据。

## 处理原则

需要跨开发/打包 origin、主窗口和辅助 WebView 保持一致的设置，应写入后端统一状态文件，由 Tauri 命令读取和更新。前端 Local Storage 可作为界面缓存，但不能作为这类设置的唯一真相源。没有证据证明基础组件有问题时，不得用原生控件替换项目设计系统组件。
