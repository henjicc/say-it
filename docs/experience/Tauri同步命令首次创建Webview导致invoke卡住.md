# Tauri 同步命令首次创建 Webview 导致 invoke 卡住

## 现象

- 前端调用打开窗口命令后一直停留在“正在打开”。
- 命令中位于窗口创建之后的状态初始化、快捷键启用等逻辑均未执行。
- 已存在的窗口可以正常显示，问题集中在首次创建 Webview。

## 根因

Windows 下从同步 `#[tauri::command]` 直接调用 `WebviewWindowBuilder::build()`，可能在等待 Tauri 主线程创建 Webview 时阻塞当前 IPC 调用，导致 `invoke` 永远不返回。

## 处理方式

- 将命令改为异步命令。
- 用 `tauri::async_runtime::spawn_blocking` 承载同步窗口创建，让 Tauri 有机会在主线程完成 Webview 构建。
- 先确认窗口成功显示，再启用依赖该窗口生命周期的快捷键或后台状态，避免创建失败后残留半启用状态。

## 验证要点

- 必须覆盖窗口尚不存在的首次打开，而不只测试重复打开。
- 验证 `invoke` 能返回、窗口可见、后续快捷键状态已启用。
- 关闭并销毁窗口后再次打开，确认重建路径同样正常。
